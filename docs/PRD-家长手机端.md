# PRD-家长手机端

## 1. Summary

家长手机端是校园智能积分兑换系统面向家长的微信小程序 Demo。它帮助家长快速查看孩子最近表现、成长报告和积分银行情况，并支持家长为孩子进行积分银行存储。

本产品不做教师管理功能，也不做奖励兑换功能。核心原则是简洁、清晰、适合年纪较大的家长使用。

## 3. Background

校园智能积分兑换系统已有教师手机端、货柜机、积分银行、成长足迹等 Demo。家长端此前没有形成清晰的独立产品定位，容易把教师端学生详情、货柜机银行、报告模块的信息直接堆到家长手机端，导致页面复杂、说明过多。

家长端用户的任务更简单：看孩子表现、看报告、看积分银行。家长不是教师，也不是终端操作员，因此不应承担管理、记录、生成报告、兑换奖励等职责。

现在需要重新定义家长端，形成清晰的页面流程和 UI 规范，避免后续继续增加多余备注、重复标签和复杂入口。

---

## 4. Objective

### 4.1 目标

让家长在微信小程序中，用最少步骤看懂孩子近期表现、报告和积分银行情况，并能完成简单的银行存储操作。

### 4.2 为什么重要

- 家长能及时了解孩子在校表现。
- 学校能更清楚地向家庭传递五育评价结果。
- 积分银行从学生端能力延伸到家校共育场景。
- 页面足够简洁，年纪较大的家长也能独立使用。

### 4.3 Key Results

| 指标 | 目标 |
|---|---|
| 首次绑定完成率 | Demo 演示中，家长能在 1 分钟内完成微信授权和孩子绑定 |
| 核心入口识别 | 家长能在 3 秒内识别底部 3 个入口：成长、报告、银行 |
| 信息阅读压力 | 每个主页面不出现非必要灰色说明，不出现重复状态标签 |
| 报告查看路径 | 家长能从报告列表进入月度报告或期末报告详情 |
| 银行操作完成 | 家长能按“选择计划 → 确认金额 → 签署存单”完成存储 |

## 5. Market Segment(s)

### 5.1 主要用户

学生家长，尤其是小学和初中阶段学生家长。

他们的核心任务不是管理系统，而是快速知道：

- 孩子最近有没有被表扬或批评。
- 学校对孩子的综合评价是什么。
- 月度报告和期末报告在哪里看。
- 孩子的积分和货币是否在增长。
- 是否需要把孩子的可用货币存入积分银行。

### 5.2 用户约束

- 部分家长年龄较大，对复杂页面、密集文字和小按钮不敏感。
- 家长不一定理解学校评价体系中的专业术语。
- 家长打开手机时通常是碎片时间，不适合阅读大量说明。
- 家长端必须避免让用户误以为自己可以替孩子兑换奖励。

---

## 6. Value Proposition(s)

### 6.1 家长获得什么

- 快速看到孩子最近表现。
- 清楚知道表扬次数、批评次数和净积分。
- 能查看月度报告和期末报告。
- 能查看可用货币、银行存款和存单。
- 能帮助孩子把可用货币存入积分银行。

### 6.2 家长避免什么痛点

- 不需要在教师端复杂信息里找孩子数据。
- 不需要理解过多系统规则。
- 不会被多余备注、说明、标签干扰。
- 不会误触兑换奖励等不属于家长端的操作。

### 6.3 产品差异点

家长端不是“信息越全越好”，而是“只保留家长当前需要的内容”。页面设计遵循强减法原则，优先服务年纪较大的家长。

---

## 7. Solution

### 7.1 UX / 页面流程

```text
微信授权登录
  ↓
绑定孩子
  ↓
成长 / 报告 / 银行
  ↓
点击顶部孩子信息，可切换孩子或继续绑定孩子
```

### 7.2 页面与功能

#### 登录页

- 显示产品主标题。
- 显示“微信授权登录”按钮。
- 不展示多余灰色说明。
- 点击后直接进入绑定孩子页面。

#### 绑定孩子页

- 字段：学校编号、学生姓名、学生学号。
- Demo 中输入任意值即可绑定。
- 不展示“输入任意信息即可体验 Demo”等说明。
- 不展示“学校编号、学生姓名、学生学号均可使用演示值”等说明。

#### 成长页

- 显示当前孩子信息。
- 显示表扬次数、批评次数、净积分。
- 支持筛选：全部、表扬、批评。
- 评价列表只展示必要信息：维度、时间、标题、分值。
- 不展示评价正文和老师信息，避免列表过重。

#### 报告页

- 显示当前孩子信息。
- 展示月度报告、期末报告列表。
- 报告列表卡片只展示报告类型和时间范围。
- 不展示报告简介、摘要或亮点。
- 点击报告进入详情页。

#### 报告详情页

- 展示报告标题、时间范围和正文内容。
- 详情页可以承载比列表页更多内容，但仍需控制段落长度。

#### 银行页

参考 demo-货柜机银行操作：

- 显示可用货币和银行存款。
- 提供“签署新存单 / 我的存单”两个标签。
- 签署新存单流程：
  - 选择存钱计划。
  - 确认存入金额。
  - 查看预计利息和到期时间。
  - 签署存单。
- 我的存单：
  - 展示活期存单、定期存单。
  - 展示本金、已得利息、进度。
  - 支持取出。

### 7.3 权限边界

家长端允许：

- 查看成长信息。
- 查看报告。
- 查看银行信息。
- 进行积分银行存储。
- 取出存单。
- 切换孩子。
- 继续绑定孩子。

家长端不允许：

- 兑换奖励。
- 兑换商品。
- 记录学生表现。
- 生成报告。
- 导出全班报告。
- 管理班级或学生。

### 7.4 UI 规则

家长端必须遵循 `PARENT_APP_UI_GUIDELINES.md`。

关键规则：

- 不增加一眼能看懂的灰色说明。
- 不展示“当前绑定的孩子”等重复标签。
- 页面标题不加常规副标题。
- 报告列表不展示内容简介。
- 底部导航固定为：成长、报告、银行。
- 主按钮高度不低于 48px。
- 页面信息密度低，适合年纪较大的家长。

### 7.5 Assumptions

- Demo 阶段绑定孩子不接真实接口，输入任意值即可生成演示数据。
- 报告详情内容使用本地 mock 数据。
- 银行计划复用现有货柜机 `BANK_CONFIG.TERMS`。
- 家长端存储行为先按本地状态变化演示，不涉及真实支付或账户安全。
- 后续如接真实接口，需要补充权限校验和数据来源说明。

## 8. Release

### 8.1 V1 范围

- 微信授权登录入口。
- 孩子绑定。
- 多孩子切换和继续绑定。
- 成长页。
- 报告页和报告详情页。
- 银行页，参考货柜机存单流程。
- 家长端 UI 规范落地。

### 8.2 暂不做

- 真实微信授权。
- 真实学校接口校验。
- 真实报告生成。
- 家长兑换奖励。
- 家长商城。
- 教师管理能力。
- 消息通知。
- 家长个人中心。

### 8.3 后续版本

- 接入真实学生绑定接口。
- 接入真实成长评价和报告数据。
- 增加家庭共育建议，但必须作为详情内容，不进入列表页。
- 增加存单风险提示和家长确认机制。
- 增加消息提醒，但不打乱底部主导航。
