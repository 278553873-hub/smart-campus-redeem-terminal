# 家长手机端样式统一设计方案

日期：2026-06-03
适用范围：家长手机端 Demo
目标：在不改变现有功能、流程和业务含义的前提下，统一家长端视觉系统与组件实现。

## 1. 背景与问题

当前家长手机端已经包含绑定孩子、成长、报告、银行、切换孩子等功能，但页面样式存在不统一问题：

- 成长、报告、银行页面的颜色、圆角、阴影、按钮风格不一致。
- 部分页面使用较多临时 Tailwind 任意值，后续维护容易继续分裂。
- 家长端混入教师端 token 类名和较强蓝色/深色按钮，视觉气质不够独立。
- 圆角和阴影缺少层级规则，部分区域显得不够专业。
- 图标使用方式不统一，页面有时偏素，有时又过度装饰。

本次目标不是重做功能，而是建立家长端专属视觉系统和组件边界，从根上解决样式不统一。

## 2. 硬约束

本方案只处理样式统一，不改变真实功能。

### 2.1 不允许做的事

- 不新增功能入口。
- 不删除现有功能。
- 不改变页面跳转流程。
- 不改变银行存单规则、金额计算、利息计算和取出逻辑。
- 不改变报告数据结构和报告内容。
- 不用概念稿里的占位内容替代真实字段。

### 2.2 必须保留的真实功能

- 绑定页：学校编号、学生姓名、学生学号、完成绑定；学校编号为空时不展示后两个字段。
- 成长页：孩子信息卡、本月净得分、表扬、待改进、预估分红总额、成长奖励、得分奖励。
- 报告页：展示 `activeChild.reports` 中的月度报告和期末报告，点击进入报告详情。
- 报告详情页：展示报告标题、孩子姓名、时间范围、摘要、亮点列表。
- 银行页：钱包、存款、签署新存单、我的存单、存钱计划列表、存入金额、单日/到期利息、到期时间、活期收益预估、签署确认、取出确认。
- 切换孩子：孩子列表、当前/切换、新增绑定。
- 底部导航：成长、报告、银行。

## 3. 视觉方向

### 3.1 整体气质

家长端应呈现高级、干净、轻科技、适合家长阅读的气质。避免儿童化、玩具感、强炫光、后台感和教师端管理感。

### 3.2 背景

- 使用极淡粉蓝、浅蓝绿的雾化背景，整体接近白色。
- 背景只营造轻氛围，不承载业务信息。
- 禁止整屏高饱和薄荷绿、蓝色或多色渐变。
- 禁止让背景抢走卡片和文字的注意力。

### 3.3 色彩

主视觉采用白色卡片 + 深色文字 + 小面积渐变 icon。

- 主背景：极淡粉蓝、浅蓝绿、接近白。
- 主要文字：接近黑色或深灰。
- 辅助文字：中性灰，不承载关键含义。
- 正向状态：青绿、绿青。
- 待改进状态：温和蜜桃橙，不使用灰色表达。
- 报告/月度类 icon：青蓝渐变。
- 期末/成长/完成类 icon：绿青渐变。
- 银行/存单类 icon：青蓝或绿青，根据操作层级选择。
- 辅助色可从“暑假研学旅行”参考图中取浅天蓝、淡奶黄、蜜桃橙、柔粉紫、亮青绿，但只用于小面积 icon、徽章、插画或状态点。

### 3.4 圆角

圆角必须分级，不能所有组件都使用大圆角。

- 页面主卡片：中等偏柔和圆角。
- 普通列表卡片：中等圆角，更利落。
- 数据小卡片：中等或略小圆角。
- 输入框和按钮：克制圆角，避免玩具感。
- 标签和状态徽章：小圆角或胶囊形，按内容长度决定。
- 底部抽屉：顶部圆角可略大，但不超过整体专业度边界。

### 3.5 阴影

- 白色卡片使用轻阴影，表现层级即可。
- 不使用厚重悬浮阴影。
- 渐变 icon 可有轻微投影，增强轻拟物质感。
- 弹窗和底部抽屉阴影可更明确，但要克制。

### 3.6 图标

- 短期使用统一渐变 icon 容器包裹现有 Lucide 图标。
- icon 容器采用青蓝、绿青、蜜桃橙等小面积渐变。
- 图标本身使用白色线性风格，保持统一。
- 不混用 emoji、随机插画和不同风格图标。
- 中期可使用 image-2 生成轻拟物 icon 资产包，但需要先定义资产清单、视角、光影、材质和尺寸规范。

### 3.7 头像

头像不属于装饰 icon。

- 学生真实上传头像优先展示。
- 未上传头像时，使用姓名姓氏第一个字作为文字头像。
- 文字头像可使用浅青蓝或浅绿背景。
- 不使用随机卡通头像或轻拟物 icon 替代学生头像。

### 3.8 按钮

- 禁止纯黑、深蓝大按钮。
- 查看类操作优先卡片化或使用轻描边按钮。
- 确认类操作使用青蓝/绿青渐变主按钮。
- 取消类操作使用浅灰或白底描边按钮。
- 危险或取出类操作不使用强烈红色，按家长端低压力原则处理。

## 4. 组件系统

本次重构应分为基础视觉层、通用组件层、业务组合层。

### 4.1 基础视觉层

建议新增家长端专属 token，覆盖：

- 背景色与背景渐变。
- 文本颜色。
- 状态色。
- icon 渐变色。
- 圆角层级。
- 阴影层级。
- 按钮状态。
- 卡片内边距和页面间距。

基础 token 不包含业务命名。

### 4.2 通用组件层

建议沉淀以下家长端组件：

- `ParentPageShell`：页面背景、滚动区、安全间距。
- `ParentHeader`：标题栏、返回按钮。
- `ParentCard`：统一白卡片、边框、圆角、阴影。
- `ParentGradientIcon`：统一渐变 icon 容器。
- `ParentChildCard`：真实头像/姓氏头像、姓名、班级、学号、切换入口。
- `ParentActionCard`：报告卡片、存单卡片等列表入口。
- `ParentMetricCard`：净得分、表扬、待改进、分红、钱包、存款等数据卡。
- `ParentBottomSheet`：存入金额、确认取出、切换孩子等底部抽屉。
- `ParentPrimaryButton`：确认类主按钮。
- `ParentSecondaryButton`：取消、轻操作、查看类按钮。

通用组件不写死业务内容，只接收标题、数值、icon、状态、按钮事件等参数。

### 4.3 业务组合层

`ParentApp.tsx` 保留业务状态、数据计算和页面流转。业务页面只负责组合组件和传入真实数据。

禁止业务页面继续散落大量自定义 `rounded-[...]`、`shadow-[...]`、渐变色和临时字体样式。

## 5. 页面映射

### 5.1 绑定页

保留真实流程：学校编号、学生姓名、学生学号、完成绑定。

样式统一方式：

- 使用家长端背景。
- 表单容器使用 `ParentCard`。
- 输入框使用统一中等圆角和清晰边框。
- 字段 label 可配小型渐变 icon，但不新增说明文字。
- 完成绑定按钮使用 `ParentPrimaryButton`。

### 5.2 成长页

保留真实内容：孩子信息、本月净得分、表扬、待改进、预估分红总额、成长奖励、得分奖励。

样式统一方式：

- 孩子信息使用 `ParentChildCard`。
- 本月净得分使用 `ParentMetricCard` 或主数据卡。
- 表扬使用绿青 icon。
- 待改进使用蜜桃橙 icon，不使用灰色。
- 预估分红总额使用独立数据卡，保留成长奖励 + 得分奖励结构。
- 不新增成长画像、高光时刻、智能状态等入口。

### 5.3 报告页

保留真实内容：月度报告、期末报告。

样式统一方式：

- 使用 `ParentActionCard`。
- 左侧为渐变 icon：月度报告青蓝，期末报告绿青。
- 中间为报告标题和时间范围。
- 右侧为浅灰箭头。
- 不展示摘要、亮点或额外入口。

### 5.4 报告详情页

保留真实内容：报告标题、孩子姓名、时间范围、摘要、亮点列表。

样式统一方式：

- 顶部使用 `ParentHeader` 返回。
- 内容使用阅读型 `ParentCard`。
- 顶部 icon 不使用黑色方块，改为统一渐变 icon。
- 摘要和亮点列表提升可读性，但不删减真实内容。

### 5.5 银行页

保留真实内容：钱包、存款、签署新存单、我的存单、存钱计划、存单列表、存入金额、利息、到期时间、活期收益预估、签署确认、取出确认。

样式统一方式：

- 钱包/存款条统一为轻量信息条或白卡。
- 签署新存单/我的存单保持 tab 功能，不改成其他入口结构。
- 存钱计划列表保留活期、7天、30天、60天、90天、180天，保留存期和日利率。
- 存入金额抽屉保留滑杆、最小值、最大可存、单日/到期利息、到期时间和活期收益预估。
- 签署确认弹窗保留计划、本金、预期利息、到期时间、我再想想、确认签署。
- 存单列表保留活期/定期、本金、已得利息、进度、取出按钮。
- 取出确认抽屉保留本金、利息、确认取出。

### 5.6 切换孩子

保留真实内容：孩子列表、当前/切换、新增绑定。

样式统一方式：

- 使用 `ParentBottomSheet`。
- 孩子项使用 `ParentChildCard` 的紧凑变体。
- 真实头像优先，未上传显示姓氏首字。
- 当前孩子使用轻状态标识，不使用过重按钮。
- 新增绑定使用主按钮或轻主按钮，按底部抽屉层级处理。

### 5.7 底部导航

保留真实入口：成长、报告、银行。

样式统一方式：

- 不沿用教师端强 3D/玻璃动效作为默认家长端风格。
- 使用白色悬浮导航，配小型统一 icon。
- 选中态使用青绿或青蓝，非选中态使用中性灰。
- 动效保持轻微，不抢页面内容。

## 6. 实施顺序

1. 建立家长端视觉 token 和通用组件。
2. 替换绑定页和报告页，验证最简单页面的组件方向。
3. 替换成长页，确保真实数据和卡片内容完全保留。
4. 替换银行页和相关弹窗/抽屉，因为交互最多，最后处理。
5. 替换切换孩子抽屉和底部导航。
6. 统一报告详情页阅读样式。
7. 跑现有测试并进行人工预览。

## 7. 验证标准

### 7.1 功能验证

- 绑定孩子流程正常。
- 学校编号为空时学生姓名和学生学号不展示。
- 绑定后进入成长页。
- 切换孩子、新增绑定正常。
- 成长页所有现有数据项保留。
- 报告列表可进入报告详情。
- 银行页 tab 切换正常。
- 存钱计划选择正常。
- 存入金额滑杆正常。
- 签署存单后钱包、存款、存单列表更新正常。
- 取出存单后钱包、存款、存单列表更新正常。

### 7.2 视觉验证

- 页面背景统一为极淡粉蓝/蓝绿雾化。
- 白色卡片、圆角、阴影统一。
- icon 风格统一，色彩小面积使用。
- 待改进使用温和蜜桃橙，不使用灰色弱化。
- 按钮不出现纯黑或深蓝大面积样式。
- 圆角不过度，不降低专业度。
- 页面不显得过素，也不因色彩过多显脏。

### 7.3 代码验证

- 业务状态和数据计算逻辑不被改动。
- 新增组件边界清晰。
- 基础组件不包含业务代码。
- 业务页面不再散落大量临时样式。
- 现有测试通过；如结构变化影响测试，补充或更新测试以保护真实功能。
