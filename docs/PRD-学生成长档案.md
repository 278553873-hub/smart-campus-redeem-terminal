# PRD：学生成长档案

## 1. 概要

学生成长档案帮助学校按照统一标准，持续记录学生在不同阶段的特点、变化和支持方式。

本产品包含三项相互解耦的能力：

- `档案设计`：面向学校档案管理员，用于新建、设计和发布本校档案模板。
- `建档任务`：面向学校档案管理员，用于按需发起任务、明确填写人并查看班级级进度。模板发布后不会自动产生任务。
- `学生成长档案`：位于学生详情页，面向班主任填写，面向有权限的教师查看。

档案内容可以由班主任手动填写，也可以由平台工作人员根据学校提供的问卷等资料，通过受控的 AI（人工智能）能力生成草稿。首期不向学校开放 AI 自助生成页面或生成按钮。

| 角色 | 负责人 | 职责 |
|---|---|---|
| 产品负责人 | 待定 | 确认产品范围、流程和验收标准 |
| 学校业务负责人 | 各试点学校指定 | 确认学校档案字段、模板和使用规则 |

| AI 交付负责人 | 平台工作人员 | 审核学校资料、执行生成、记录消耗和交付草稿 |

## 3. 背景

### 3.1 当前问题

学校已经通过教师评价、家长问卷、学生访谈等方式积累了大量学生信息，但这些信息通常存在以下问题：

1. 信息分散在问卷、评价记录、表格和老师个人经验中。
2. 不同学校、年级和班级关注的档案字段不一致。
3. 学生更换班主任或任课教师后，新老师需要重新了解学生。
4. 传统档案更像一次性登记表，难以体现学生在不同阶段的变化。
5. 模板发布、任务发起和学生档案内容混在一起时，容易误把“设计标准”理解为“要求全校立即填写”。
6. 如果直接向学校开放批量 AI 生成，可能出现重复操作、无效生成和成本失控。

### 3.2 产品机会

学校首先需要统一“记录什么”，再按实际需要决定“是否发起、由谁填写、何时完成”。因此，模板设计、建档任务和学生个人档案必须分别授权、分别进入。学校可以只发布模板，不发起任务；教师也可以在有权限时从学生详情单独新建档案。

### 3.3 定制学校调研结论

根据《增值评价体系设计方案》，该校的首要诉求不是建设复杂的自由表单，而是将一年级初始成长档案结构化。档案需要形成以下五个维度的入学画像：

1. 学业基础。
2. 兴趣偏好。
3. 认知特点。
4. 交往风格。
5. 性格与家庭。

除五维画像外，模板还需要记录三方优先发展目标、六维初始光芒定位，以及兴趣激发度、胜任感、归属感三个内驱力信号。

文档中的日常行为指标、成长瞬间、周月分析和学期报告属于档案的证据来源及展示分析，不直接转换为档案模板字段，避免模板过长、班主任重复填写。

### 3.4 产品定位

学生成长档案是一套学校级的学生信息沉淀与教师交接机制。

`档案设计`不是自由表单工具，也不是学生档案内容生产工具。它的核心定位是：

> 让学校管理层根据本校育人理念和学生管理要求，设计并发布统一的学生档案模板及字段。

`建档任务`的核心定位是：

> 当学校确有统一建档需要时，将指定模板、学生范围、填写责任人、截止时间和提醒组成一次可追踪任务。任务不是模板发布的必经步骤。

`学生成长档案`的核心定位是：

> 让班主任按照学校模板记录学生阶段性情况，并让后续教师快速了解学生的优势、变化、关注事项和有效支持方式。

## 4. 产品目标

### 4.1 业务目标

1. 帮助学校建立统一、可持续使用的学生档案标准。
2. 让学校可以选择只发布模板，或另行发起一次建档任务，避免功能强绑定。
3. 让班主任或被明确指定的任课教师收到待办并完成填写。
4. 形成按阶段保存的档案快照，体现学生成长变化。
5. 让新班主任和当前任课教师快速了解学生情况。
6. 在满足学校定制需求的同时，严格控制 AI 使用范围和生成成本。

### 4.2 用户目标

学校管理层需要：

- 明确本校学生档案应该记录哪些字段。
- 使用分组表达本校档案的维度和育人框架。
- 为不同年级或阶段设置不同模板。
- 保证已经发布和使用的模板不会被随意改动。
- 在确有需要时发起建档任务，并明确每个班级的填写责任人。
- 只查看任务的班级级进度和异常，不浏览学生个人档案内容。

班主任需要：

- 在学生详情中直接填写档案。
- 保存草稿并继续填写。
- 查看学生历史档案和上一阶段内容。
- 确认成档后保留历史版本。
- 在“我的”中及时看到分配给自己的建档任务。

被指定的任课教师需要：

- 仅在被明确指派后获得对应班级、对应任务的临时填写权限。
- 从“我的 → 待完成建档”进入任务并完成填写。
- 任务结束或任教关系失效后失去临时填写权限。

有查看权限的教师需要：

- 快速看到学生最新情况。
- 了解学生相较上一阶段发生的变化。
- 了解对学生有效的沟通、教学和支持方式。

平台工作人员需要：

- 在学校提出明确需求后，受控执行 AI 生成。
- 掌握每次生成的学校、模板、学生数量和资料来源。
- 控制生成次数和消耗。

### 4.3 成功指标

首批试点学校启用后的一个完整档案周期内：

| 指标 | 目标 |
|---|---|
| 学校模板发布成功率 | 100% |
| 班主任档案草稿保存成功率 | 99%以上 |
| 已成档内容被模板更新误改的数量 | 0 |
| AI 生成结果由班主任确认后成档的比例 | 100% |
| 未经平台工作人员授权的 AI 生成次数 | 0 |
| AI 生成任务的学校、模板、数量和消耗记录完整率 | 100% |
| 新接班教师完成一次学生档案查看的平均时间 | 2分钟以内 |
| 建档任务责任人明确率 | 100% |
| 建档任务发起后到达填写人待办的成功率 | 99%以上 |
| 管理者从任务页打开学生个人档案的次数 | 0 |

### 4.4 非目标

首期不解决以下问题：

- 不建设学校自主操作的 AI 生成中心。
- 不建设面向学校的批量 AI 生成按钮。
- 不展示跨任务汇总的“全校总成档率”或“学生已成档总数”。
- 不把模板发布自动转换为建档任务。
- 不在档案设计或任务管理页面提供学生个人档案查询。
- 不在档案设计页面收集家长问卷或学生自评。
- 不把日常评价指标、周月分析或学期报告整体复制为档案字段。
- 不向个人版、协作版教师开放档案设计。
- 不支持区级或跨学校查看学生个人档案。

## 5. 目标用户

### 5.1 核心用户：学校档案管理员

学校档案管理员是由学校指定的管理角色，可以是校长、分管副校长、德育处、学生发展中心或其他授权人员。

核心任务：

- 使用推荐模板。
- 设计校本模板。
- 配置档案字段。
- 设置模板适用范围。
- 发布、停用和升级模板版本。
- 按需发起建档任务，并查看班级级进度与责任人异常。

档案设计入口只向学校档案管理员展示。年级组长、班主任或任课教师不会因为拥有任教关系而自动获得该入口。

### 5.2 核心用户：班主任

班主任是学生档案内容的默认维护者和最终确认人。

核心任务：

- 从学生详情进入学生成长档案。
- 根据学校模板新建阶段档案。
- 手动填写或检查 AI 生成草稿。
- 保存草稿。
- 确认成档。
- 对已成档内容申请更正。

班主任不进入档案设计模块，不负责修改学校模板。班主任被分配建档任务时，从“我的 → 待完成建档”进入自己的任务；没有任务时，仍可按权限从学生详情单独新建档案。

### 5.3 补充用户：被指定的任课教师

学校确有特殊安排时，可以按班级指定一名当前有任教关系的教师填写某次建档任务。

核心任务：

- 接收本次建档任务提醒。
- 填写任务范围内学生的档案草稿。
- 保存草稿并提交给班主任或按学校规则确认成档。

指定权限只对本次任务和对应班级有效。不得选择与该班没有任教关系的教师。

### 5.4 使用用户：当前任课教师和新接班教师

核心任务：

- 查看自己当前任教班级学生的档案。
- 查看最新档案、成长变化和有效支持方式。
- 按需查看历史阶段档案。

教师任教关系结束后，自动失去对应学生档案的查看权限。

### 5.5 内部用户：平台工作人员

核心任务：

- 接收学校提出的 AI 档案生成需求。
- 检查学校提供的问卷、访谈或其他资料。
- 确认模板、学生范围和预计消耗。
- 通过内部工具或受控流程触发生成。
- 将生成结果写入学生档案草稿。
- 记录生成过程和消耗。

### 5.6 信息来源用户：家长和学生

家长和学生可以通过问卷、访谈等方式提供信息，但不是教师手机端档案设计功能的直接用户。

## 6. 用户价值

### 6.1 对学校管理层

- 将学校育人理念落实为统一的学生档案字段。
- 不同班级使用同一套档案标准。
- 可以为入学、学期和毕业等阶段设计不同模板。
- 模板更新不会破坏历史档案。

### 6.2 对班主任

- 在学生详情中就近填写，减少页面跳转。
- 可以参考上一阶段内容，避免重复了解和重复填写。
- AI 生成时只需要检查和修改草稿，不需要从空白开始。
- 已成档内容具有明确的时间和责任人。

### 6.3 对新老师和任课教师

- 快速了解学生的优势、兴趣、关注事项和支持方式。
- 能看到学生前后变化，而不是只看到静态标签。
- 减少因教师更换造成的信息中断。

### 6.4 对平台

- 通过模板能力满足不同学校的定制需求。
- 通过人工审核和内部触发控制 AI 消耗。
- 避免学校无目的、重复或大批量生成档案。

## 7. 产品方案

### 7.1 产品结构

#### 学校管理层路径

```text
教师手机端 → 我的 → 更多工具 → 档案管理
→ 档案设计 Tab：新建、编辑和发布档案模板
→ 建档任务 Tab：按需发起任务并查看班级级进度
```

该入口只面向学校档案管理员。两个 Tab 不展示跨任务总完成率，也不提供学生档案查询。

#### 建档任务填写路径

```text
教师手机端 → 我的 → 待完成建档 → 任务 → 待填写学生
```

任务默认分配给各班当前班主任。学校按班级指定当前任课教师时，该教师仅获得本次任务的填写与确认权限。

#### 班主任填写路径

```text
教师手机端 → 班级 → 学生详情 → 学生成长档案
```

#### 任课教师查看路径

```text
教师手机端 → 班级 → 学生详情 → 学生成长档案
```

任课教师默认只能查看当前任教关系下的学生档案，不能填写、确认或更正。被明确指定为某次任务责任人时，只能填写和确认该任务范围内的档案。

#### AI 内部交付路径

```text
学校提出需求并提供资料
→ 平台工作人员审核需求
→ 内部触发 AI 生成
→ 生成学生档案草稿
→ 班主任检查和修改
→ 班主任确认成档
```

学校端不提供 AI 生成页面。

### 7.2 功能一：学校档案模板管理

#### 7.2.1 模板列表

档案管理首页使用两个并列 Tab：`档案设计`和`建档任务`。默认进入`档案设计`。

`档案设计`顶部提供“新建档案”主操作。点击后先进入“选择模板”，选择推荐模板后创建一份校本模板草稿，再进入结构编辑。界面称“新建档案”，数据对象仍是“档案模板草稿”，不会为学生创建个人档案，也不会自动发起任务。

模板列表分为：

- 推荐模板：由平台提供，学校不能直接修改。
- 校本模板：学校复制推荐模板或自行创建的模板。

模板状态：

- 草稿：仅学校档案管理员可见，可以编辑。
- 已发布：可以被班主任用于新建学生档案。
- 已停用：不能再用于新建档案，但历史档案仍然保留。

#### 7.2.2 推荐模板

首期提供三类推荐模板：

1. 入学基线档案。
2. 学期成长档案。
3. 转学、升学或毕业档案。

学校可以直接使用，也可以复制后形成校本模板。

#### 7.2.3 模板基础设置

学校档案管理员可以设置：

- 模板名称。
- 档案阶段：入学、学期、转衔或学校自定义。
- 适用年级。
- 是否启用。

模板基础设置不包含任务、截止日期和完成进度。以上内容只在`建档任务`中设置。

#### 7.2.4 字段设计

支持以下字段能力：

- 新增、重命名、排序和删除字段分组。
- 新增字段。
- 删除不适用的推荐字段或学校自定义字段。
- 调整字段顺序。
- 修改字段名称。
- 将字段移动到其他分组。
- 设置是否必填。
- 设置字段类型。
- 编辑单选或多选选项。
- 设置字段是否用于跨阶段成长对比。
- 预览班主任填写效果。

首期字段类型：

- 简短文字。
- 多行文字。
- 单选。
- 多选。

首期不支持复杂条件跳转、计算公式或自由拖拽布局。

#### 7.2.5 字段分组

字段分组用于表达学校的档案结构，例如“学业基础”“认知特点”和“内驱力特征”。

- 每个字段必须归属一个分组。
- 学校可以调整分组名称和顺序。
- 班主任填写时按分组展示字段，默认展开第一个未完成分组。
- 已完成或暂不处理的分组可以折叠，避免长表单一次性铺满手机页面。
- 空分组可以删除；包含字段的分组需要先移动或删除字段。

#### 7.2.6 推荐字段与稳定标识

平台推荐模板默认包含以下建议用于教师交接的字段：

- 优势特点。
- 兴趣倾向。
- 学习习惯。
- 情绪状态。
- 同伴交往。
- 当前关注。
- 有效支持方式。
- 阶段目标。

以上字段是推荐项，不是全校强制字段。学校复制为校本模板后，可以修改、停用或删除。

学校将某个字段设置为“用于成长对比”后，该字段保留稳定标识。后续模板版本即使修改显示名称，只要稳定标识不变，系统仍可以比较不同阶段的内容。

#### 7.2.7 一年级初始档案参考模板

根据定制学校调研，首期提供“一年级初始成长档案”参考模板：

| 分组 | 字段 | 类型 |
|---|---|---|
| 学业基础 | 基础认知、专注习惯、提问与任务 | 单选 |
| 兴趣偏好 | 兴趣倾向 | 多选 |
| 兴趣偏好 | 动手创意 | 单选 |
| 认知特点 | 学习方式 | 多选 |
| 认知特点 | 问题解决 | 单选 |
| 交往风格 | 帮助分享、规则礼貌、冲突处理 | 单选 |
| 性格与家庭 | 情绪稳定性、运动活力、主要照顾人、家庭陪伴时间 | 单选 |
| 优先发展目标 | 家长期望、学生自选、教师建议 | 多选，选项为求真、从善、尚美、学活、乐健、悦群 |
| 内驱力特征 | 当前突出内驱力信号 | 单选，选项为兴趣激发、胜任感、归属感、尚不明确 |
| 内驱力特征 | 初始光芒定位、下一阶段优先关注方向 | 多选 |
| 建档确认 | 家长确认 | 单选，非必填 |
| 教师交接摘要 | 优势特点、当前关注、有效支持方式、阶段目标 | 多行文字 |

学生姓名、班级、性别、建档日期、建档教师、健康提醒和过敏史作为系统信息自动带出，不由学校重复创建字段，不要求班主任重复填写。

#### 7.2.8 其他阶段参考字段

入学档案建议包含：

- 家庭观察。
- 入学准备。
- 初始兴趣。
- 适应情况。
- 家长期望。

学期成长档案建议包含：

- 本学期变化。
- 代表性证据。
- 目标达成情况。
- 新发现的优势。
- 下阶段策略。

转衔档案建议包含：

- 长期成长概览。
- 最有效的教育方式。
- 仍需支持事项。
- 重要变化。
- 交接建议。

#### 7.2.9 模板发布和版本

- 已发布模板不能直接修改。
- 修改已发布模板时，系统创建新的草稿版本。
- 新版本发布后，只影响之后新建的学生档案。
- 已有档案草稿继续使用创建时绑定的模板版本。
- 已成档内容不会因模板修改而变化。

### 7.3 功能二：按需发起建档任务

#### 7.3.1 发起前提

- 只有学校档案管理员可以发起任务。
- 只能选择已发布且适用于目标年级的校本模板。
- 模板发布不会自动进入任务流程。
- 学校可以长期只使用模板和学生详情手动建档，不发起任何任务。

#### 7.3.2 任务设置

发起任务必须设置：

- 档案模板及版本。
- 所属周期。
- 目标年级和班级。
- 每个班级的填写责任人。
- 截止日期。
- 提醒规则。

任务名称由周期和档案阶段生成，允许管理员修改。任务只引用模板版本，不复制或改写模板。

#### 7.3.3 填写责任人

每个班级必须且只能有一名任务责任人：

1. 默认选择该班当前班主任。
2. 特殊情况下，可以改为一名当前对该班有任教关系的教师。
3. 没有班主任且未指定任课教师的班级标记为“未分配”，任务不能发起。
4. 责任人获得本次任务范围内的填写、保存草稿和确认成档权限，不获得其他班级或其他任务的权限。
5. 管理员只能查看责任人、班级完成数、待完成数和异常状态，不能从任务页打开学生个人档案。

#### 7.3.4 提醒与入口

任务发起后，系统执行以下提醒：

- 立即写入责任教师“我的 → 待完成建档”，并在“我的”底部导航显示合并角标。
- 默认在截止前 3 天、截止前 1 天提醒仍未完成的责任教师。
- 逾期后默认每天提醒一次，直到完成、任务撤回或管理员关闭逾期提醒。
- 已完成任务不再提醒。
- 提醒点击后进入该教师被分配的任务，不进入档案设计。

首期提醒使用应用内待办和消息能力；是否增加微信服务通知由学校授权和平台能力决定。

#### 7.3.5 任务进度

管理员的任务详情按班级展示：

- 班级名称。
- 责任人和责任角色。
- 已完成人数、总人数。
- 状态：未分配、未开始、进行中、已完成、已逾期。

不展示学生姓名列表，不提供学生搜索、档案内容预览、下载或转发。

责任教师的任务详情只展示自己负责班级的学生填写状态。未完成学生可以进入填写，已成档学生只显示完成状态；查看已成档内容需要回到班级学生详情。

#### 7.3.6 状态与操作

任务状态包括：草稿、进行中、已完成、已撤回。

- 草稿：管理员可以修改范围、责任人、截止日期和提醒。
- 进行中：可以调整未开始班级的责任人和截止日期；已产生草稿的班级调整责任人时需要二次确认并保留交接记录。
- 已完成：所有目标学生已成档后自动完成，也可以查看班级级结果。
- 已撤回：不再创建新草稿和发送提醒；已有草稿按学校选择保留或作废，已成档内容不受影响。

### 7.4 功能三：教师手动填写档案

#### 7.4.1 学生详情入口

学生详情新增“学生成长档案”入口。

入口展示：

- 最新一份档案的阶段和时间。
- 历史档案数量。
- 新建档案入口。

#### 7.4.2 新建档案

班主任点击“新建档案”后：

1. 系统展示适用于当前学生年级的已发布模板。
2. 班主任选择模板。
3. 系统记录模板名称和版本。
4. 进入档案填写页。

同一学生、同一档案阶段和同一周期已经存在草稿时，系统优先打开已有草稿，避免重复创建。

#### 7.4.3 填写档案

- 顶部紧凑展示系统自动带出的学生和建档信息。
- 按学校模板的分组和字段顺序展示。
- 默认展开第一个未完成分组，其他分组渐进展开。
- 设置为成长对比的字段可以展示上一阶段内容，供班主任参考。
- 支持文字、单选和多选填写。
- 支持保存草稿。
- 草稿仅当前填写责任人可见。学校档案管理员只能看到草稿状态，不能查看草稿内容；同一人同时具备当前任教权限时，按教师身份访问。

#### 7.4.4 确认成档

- 必填字段完成后才允许确认成档。
- 确认成档前展示二次确认。
- 成档后记录模板版本、任务责任人、责任角色、成档时间和内容来源。
- 已成档内容不能直接编辑。

#### 7.4.5 更正档案

- 原成档责任人或当前班主任可以对已成档内容申请更正。
- 系统基于原档案生成更正草稿。
- 更正完成后形成新的修订版本。
- 原档案继续保留，不能被覆盖或删除。

### 7.5 功能四：AI 受控生成档案草稿

#### 7.5.1 首期原则

- 学校端不展示 AI 生成入口。
- 学校不能自行批量生成。
- 只有平台工作人员可以触发生成。
- AI 生成内容只能进入草稿状态。
- 当前档案责任教师必须检查并确认后才能成档。

#### 7.5.2 可使用的资料

学校可以按实际需求提供：

- 家长问卷。
- 学生自评或访谈。
- 教师观察表。
- 日常评价记录。
- 学校认可的其他结构化资料。

#### 7.5.3 生成前检查

平台工作人员需要确认：

- 学校名称。
- 使用的档案模板和版本。
- 学生范围和数量。
- 资料来源。
- 资料是否完整。
- 预计生成次数和预计消耗。
- 是否获得学校授权。

#### 7.5.4 生成规则

- 只能生成模板中存在的字段。
- 不得自行增加模板外字段。
- 资料中没有依据的内容保持为空，不能编造。
- 生成内容需要保留资料来源。
- 一名学生生成失败不能影响其他学生。
- 重复提交相同资料时，需要阻止重复生成或提示工作人员确认。

#### 7.5.5 生成结果

生成结果写入学生档案草稿，当前档案责任教师可以：

- 查看生成内容。
- 修改内容。
- 补充空白字段。
- 放弃本次草稿。
- 确认成档。

#### 7.5.6 AI 成本控制

每次生成必须记录：

- 操作人员。
- 学校。
- 模板和版本。
- 学生数量。
- 资料来源。
- 开始和结束时间。
- 成功、失败和跳过数量。
- 实际消耗。

平台可以为学校设置内部预算或生成上限。达到限制后，工作人员不能继续生成，必须由负责人确认。

### 7.6 功能五：档案查看与教师交接

#### 7.6.1 最新档案

学生成长档案首页优先展示：

- 优势特点。
- 近期变化。
- 当前关注。
- 有效支持方式。
- 阶段目标。

#### 7.6.2 成长变化

对于具有相同稳定标识的核心字段，系统可以展示上一阶段和当前阶段内容，帮助老师了解变化。

#### 7.6.3 历史档案

按时间倒序展示：

- 档案阶段。
- 所属周期。
- 模板和版本。
- 成档时间。
- 成档班主任。

#### 7.6.4 长期底档

以下信息不随阶段档案重复保存：

- 学生身份信息。
- 班级经历。
- 健康提醒。
- 过敏史。
- 家庭联系人。

长期底档发生变化时即时更新，并保留修改记录。

### 7.7 权限与审计

| 能力 | 学校档案管理员 | 班主任 | 被指定任课教师 | 其他当前任课教师 | 平台工作人员 |
|---|---|---|---|---|---|
| 设计和发布模板 | 可以 | 不可以 | 不可以 | 不可以 | 不可以 |
| 发起和管理建档任务 | 可以 | 不可以 | 不可以 | 不可以 | 不可以 |
| 查看任务班级级进度 | 可以 | 仅本人任务 | 仅本人任务 | 不可以 | 不可以 |
| 新建和填写档案 | 不可以 | 可以 | 仅指定任务 | 不可以 | 仅写入 AI 草稿 |
| 查看草稿内容 | 不可以 | 本人负责范围 | 仅指定任务 | 不可以 | 仅在交付范围内 |
| 确认成档 | 不可以 | 可以 | 仅指定任务 | 不可以 | 不可以 |
| 查看完整档案 | 仅同时具备任教关系时 | 可以 | 当前任教范围 | 当前任教范围 | 仅在交付范围内 |
| 申请更正 | 不可以 | 可以 | 仅本人原成档内容 | 不可以 | 不可以 |
| 触发 AI 生成 | 不可以 | 不可以 | 不可以 | 不可以 | 可以 |

审计范围：

- 模板创建、修改、发布和停用。
- 任务创建、责任人调整、提醒、撤回和状态变化。
- 学生档案新建、保存、成档和更正。
- 完整档案查看。
- 长期底档修改。
- AI 生成、失败、重试和消耗。

首期默认不提供档案下载和转发能力。

### 7.8 核心数据对象

#### 档案模板

- 模板编号。
- 学校编号。
- 模板名称。
- 档案阶段。
- 适用年级。
- 状态。
- 当前版本。

#### 模板字段

- 字段编号。
- 稳定字段标识。
- 所属分组编号。
- 显示名称。
- 字段类型。
- 是否必填。
- 选项。
- 排序。

#### 模板字段分组

- 分组编号。
- 分组名称。
- 分组说明，选填。
- 排序。

#### 学生档案草稿

- 学生编号。
- 模板和版本。
- 档案阶段和周期。
- 字段内容。
- 内容来源：责任教师填写或 AI 生成。
- 创建人和更新时间。

#### 建档任务

- 任务编号、学校和状态。
- 档案模板及固定版本。
- 周期、目标年级和班级。
- 截止日期和提醒规则。
- 创建人、创建时间、撤回人和撤回时间。

#### 班级任务指派

- 任务编号、班级编号和班级名称。
- 责任教师编号、姓名和角色。
- 已完成、待完成和异常数量。
- 指派和调整记录。

#### 已成档快照

- 草稿的完整内容副本。
- 成档班主任。
- 成档时间。
- 修订关系。

#### AI 生成任务

- 学校、模板和学生范围。
- 资料来源。
- 操作人员。
- 状态和结果。
- 消耗记录。

### 7.9 关键业务规则

1. 学校必须先发布模板，班主任才能新建档案。
2. 已发布模板通过新版本修改，不能直接覆盖。
3. 草稿始终绑定创建时的模板版本。
4. 已成档内容不可直接修改。
5. AI 内容只能生成草稿，不能直接成档。
6. 班主任是默认档案责任人；被明确指定的任课教师是该次任务范围内的档案责任人和最终确认人。
7. AI 没有资料依据时必须留空。
8. 任教关系结束后，教师失去对应学生档案权限。
9. 平台工作人员只能查看和处理明确交付范围内的数据。
10. 所有查看、修改、成档和生成操作必须留痕。
11. 推荐字段不是强制字段，校本模板可以删除不适用的推荐字段。
12. 系统信息由学生资料和当前登录信息自动带出，不作为班主任手工填写项。
13. 删除包含字段的分组前，必须先移动或删除分组内字段。
14. 模板发布与任务发起相互独立，任何一方状态变化都不能自动触发另一方操作。
15. 每个任务班级必须有且只有一名责任人；责任人必须与目标班级存在有效任教关系。
16. 档案管理员身份不自动获得学生档案内容查看权。

### 7.10 异常与边界情况

| 场景 | 处理方式 |
|---|---|
| 学校没有已发布模板 | 学生详情不展示“新建档案”，提示联系学校档案管理员 |
| 模板在草稿填写期间发布新版本 | 当前草稿继续使用原版本 |
| 模板被停用 | 已有草稿可以继续完成，不能再新建 |
| 同一周期重复新建 | 优先打开已有草稿 |
| 同一学生同时存在入学档案和学期档案 | 作为两份独立档案处理，不合并进度 |
| 同一模板、周期和学生被重复加入任务 | 阻止重复创建，定位到已有草稿或已成档记录 |
| 班级没有班主任 | 必须指定一名有任教关系的教师，否则任务不能发起 |
| 任务责任教师在任务中途离职或解除任教关系 | 立即暂停其填写权限，班级标记“待重新分配”，停止向原教师提醒 |
| 任务期间更换班主任 | 未开始内容自动建议转给新班主任；已有草稿需管理员确认交接，保留原填写人记录 |
| 任务逾期 | 保留填写能力并标记逾期，按提醒设置继续提醒，不自动成档 |
| 任务被撤回 | 停止提醒和新建；草稿按撤回选择保留或作废，已成档内容保留 |
| AI 资料缺失 | 对应字段留空并标记缺少来源 |
| 单个学生 AI 生成失败 | 记录失败原因，其他学生继续生成 |
| 班主任更换 | 新班主任获得当前班级学生档案权限 |
| 学生转班或离校 | 按新的组织关系调整权限，历史档案保留 |
| 已成档内容错误 | 通过更正草稿生成修订版本 |

### 7.11 产品假设

以下假设需要通过试点验证：

1. 学校愿意指定少量档案管理员维护模板。
2. 学校只会在确有统一收集需要时发起建档任务，不会把所有已发布模板都转为任务。
3. 责任教师需要同时保留“我的待办”和“学生详情”两条入口，才能兼顾集中完成与就近维护。
4. 每学期或关键阶段形成一次档案，能够兼顾成长性和工作量。
5. AI 由平台工作人员触发，可以在首期有效控制成本。
6. 平台推荐模板能够覆盖多数学校的基础需要。

## 8. 发布计划

### 8.1 第一阶段：模板、手动档案与受控 AI 交付

包含：

- 学校档案管理员权限。
- 推荐模板和校本模板。
- 字段分组、字段设计和模板预览。
- 一年级初始成长档案参考模板。
- 模板发布、停用和版本管理。
- `档案设计 / 建档任务`双 Tab 信息架构。
- “新建档案”选择推荐模板并生成校本模板草稿。
- 可选建档任务、逐班责任人、截止日期和应用内提醒。
- 管理员班级级任务进度，以及责任教师“我的 → 待完成建档”。
- 学生详情档案入口。
- 班主任或被指定任课教师填写、保存草稿和确认成档。
- 历史档案、成长变化和长期底档查看。
- 更正机制和操作日志。
- 学校提出需求后，由平台工作人员审核资料和生成范围。
- 平台工作人员通过内部受控方式生成档案草稿。
- AI 生成结果由当前档案责任教师检查、修改和确认成档。
- AI 生成学校、模板、学生数量和消耗记录。

首期不建设学校端 AI 生成页面，也不向学校开放生成按钮。

### 8.2 第二阶段：内部 AI 交付工具标准化

包含：

- 面向平台工作人员的内部生成工作台。
- 问卷等资料的标准化接入和校验。
- 批量生成、失败重试和结果检查。
- 更完整的生成成本、预算和使用统计。
- AI 交付任务的内部审批和责任追踪。

第二阶段仍不向学校开放 AI 自助生成页面，只提高平台内部交付效率。

### 8.3 后续候选能力

根据试点结果再决定是否建设：

- 对接学校统一消息中心和经授权的微信服务通知。
- 支持管理员按班级批量调整责任人和更灵活的提醒节奏。
- 更丰富的字段类型和模板规则。
- 家长端档案发布与确认。
- 在明确预算、权限和审核机制后，有限开放学校自助 AI 生成。

### 8.4 首期验收标准

1. 只有学校档案管理员能看到档案管理入口，首页使用`档案设计 / 建档任务`两个 Tab。
2. 页面不展示跨任务总完成率、学生已成档总数或学生档案查询。
3. “新建档案”先选择推荐模板，再生成可编辑的校本模板草稿。
4. 学校可以使用推荐模板或复制形成校本模板。
5. 校本模板可以配置分组、字段、必填、选项、排序、成长对比和适用范围。
6. 已发布模板只能通过新版本修改，发布后不会自动发起任务。
7. 建档任务必须明确模板、周期、班级、逐班责任人、截止日期和提醒规则。
8. 默认责任人是当前班主任；指定任课教师时必须校验其有效任教关系。
9. 管理员任务页只展示班级级进度和异常，不能查看学生名单或打开学生档案。
10. 责任教师可以在“我的 → 待完成建档”收到任务，并只查看自己负责班级的待填写学生。
11. 班主任可以从学生详情单独新建档案，不依赖建档任务。
12. 责任教师可以保存草稿和确认成档，已成档内容不能直接修改。
13. 普通任课教师可以查看当前任教学生档案，但不能填写；被指定时只获得对应任务的临时填写权限。
14. 学校端不存在 AI 生成按钮或生成页面，平台工作人员生成的内容只能进入草稿。
15. 任务指派与调整、提醒、AI 生成、档案查看、成档和更正均保留操作记录。
16. 一年级初始档案模板完整包含五维画像、三方发展目标、六维光芒定位和内驱力字段。
17. 填写页按分组渐进展开，姓名、班级、性别、建档信息由系统自动带出。
