# PRD：学生成长档案

## 1. 概要

学生成长档案用于帮助学校统一“记录什么”，并让老师在自己有权限的学生详情中按需建档。

首期采用轻量化方案，只包含两项相互解耦的能力：

- `档案设计`：学校领导或学校档案管理员创建档案，并管理启用、禁用状态。
- `学生成长档案`：所有能进入该学生详情的老师，都可以选择学校已启用的档案进行填写、保存和成档。

系统不提供建档任务、任务发布、填写人指派、截止时间、提醒或任务进度。学校启用档案，只代表老师可以使用，不代表学校要求老师必须填写。

## 3. 背景

### 3.1 当前问题

学校已经通过评价记录、家长问卷、学生访谈和老师观察积累了大量学生信息，但仍存在以下问题：

1. 信息分散，换老师后需要重新了解学生。
2. 不同学校和年级需要记录的内容不同。
3. 一个学生可能同时有入学档案、学期档案、转衔档案等多份档案，不能用单一“是否已成档”描述。
4. 学校设计好档案后，可能因工作安排或信息安全要求，不强制老师填写。
5. 如果把档案设计、任务发布、学生查询和填写进度放在同一处，会增加管理成本，也会让“启用档案”被误解为“向全校下发任务”。

### 3.2 调整原因

学校领导确认，当前最需要的是一套简单、可控的校本档案标准，而不是一套任务管理系统。

因此，产品调整为：

- 学校只负责设计档案和启用、禁用。
- 老师只在学生详情中按需使用档案。
- 班级和学生权限继续沿用现有系统，不在档案模块内重复配置。
- 档案设计者不因管理模板而自动获得学生档案内容查看权。

### 3.3 产品定位

`档案设计`的定位是：

> 让学校建立可复用的档案结构，并决定哪些档案当前可供老师使用。

`学生成长档案`的定位是：

> 让有学生访问权限的老师，在学生详情中按需记录学生成长信息，并保留历史档案。

档案设计与学生档案内容完全解耦：启用不会自动创建学生档案，不会生成待办，也不会产生完成率。

## 4. 目标

### 4.1 业务目标

1. 帮助学校建立统一、可持续使用的档案结构。
2. 用启用、禁用控制档案是否可用于新建，降低管理成本。
3. 让老师在学生详情就近建档，避免从管理页面查找学生。
4. 支持一个学生拥有多份不同用途的档案。
5. 保证模板状态变化不破坏已有草稿和历史档案。
6. 保证档案设计权限与学生内容查看权限相互独立。

### 4.2 用户目标

学校领导或学校档案管理员需要：

- 完全新建一份档案，或从推荐模板创建。
- 设置档案名称、适用年级、分组和字段。
- 保存未完成的设计草稿。
- 启用可供老师使用的档案。
- 禁用暂时不再允许新建的档案。
- 不查看学生名单、学生档案和全校成档统计。

老师需要：

- 在自己有权限的学生详情中进入学生成长档案。
- 选择适用于当前学生年级的已启用档案。
- 保存草稿并继续填写。
- 确认成档。
- 查看该学生的历史档案。

### 4.3 成功指标

| 指标 | 目标 |
|---|---|
| 档案设计草稿保存成功率 | 99%以上 |
| 合格档案启用成功率 | 100% |
| 学生档案草稿保存成功率 | 99%以上 |
| 已成档内容被模板状态变化误改的数量 | 0 |
| 禁用后仍能新建该档案的次数 | 0 |
| 禁用导致已有草稿或历史档案丢失的数量 | 0 |
| 无学生权限的教师访问档案成功次数 | 0 |

### 4.4 非目标

首期不建设以下能力：

- 不发布建档任务。
- 不指定班主任或任课教师作为填写责任人。
- 不设置截止时间、提醒规则或任务进度。
- 不在“我的”中增加建档待办或角标。
- 不在档案设计中查询学生或查看学生档案。
- 不展示全校总成档率、已成档学生数等汇总统计。
- 不因为启用档案而批量创建学生草稿。
- 不在档案模块内重新配置老师的班级权限。
- 不向个人版、协作版开放学校级档案设计。
- 不默认开放档案下载和转发。

## 5. 目标用户

### 5.1 学校领导或学校档案管理员

核心任务：

- 创建校本档案。
- 设计字段和分组。
- 启用、禁用档案。

约束：

- 档案设计入口由学校空间角色控制。
- 管理档案设计不等于拥有学生档案内容权限。
- 页面不展示学生名单和学生档案内容。

### 5.2 有学生访问权限的老师

包括班主任、当前任课教师，以及现有系统授权的其他老师。

核心任务：

- 从班级进入学生详情。
- 新建、填写和保存档案。
- 继续已有草稿。
- 确认成档并查看历史。

约束：

- 老师能访问哪些班级和学生，完全沿用现有系统权限。
- 档案模块不增加一套新的班级授权关系。
- 老师失去班级或学生访问权限后，同时失去对应档案访问权限。

### 5.3 平台工作人员

在学校明确授权后，可以使用内部工具将问卷等资料生成档案草稿。平台工作人员不能直接确认成档，生成结果仍需有学生访问权限的老师检查。

## 6. 用户价值

### 6.1 对学校

- 只管理档案标准，不承担任务分配和催办成本。
- 可以为入学、学期、毕业等场景分别创建档案。
- 可以先设计和启用，是否填写由老师结合实际需要决定。
- 禁用只影响以后新建，不破坏已有资料。

### 6.2 对老师

- 在学生详情就近完成建档，不需要从管理页面找学生。
- 可以选择当前真正需要的档案，不被统一任务打断。
- 草稿可随时保存，历史档案可持续查看。
- 可参考上一份历史档案内容，减少重复了解和重复填写。

### 6.3 对学生信息安全

- 模板设计权与学生内容查看权分离。
- 老师只处理自己原本就有权限访问的学生。
- 启用档案不会批量复制或暴露学生信息。
- 查看、保存和成档保留操作记录。

## 7. 产品方案

### 7.1 产品结构

#### 学校管理路径

```text
教师手机端 → 我的 → 更多工具 → 档案设计
→ 新建档案
→ 完全新建 / 从模板创建
→ 编辑并保存草稿
→ 启用 / 禁用
```

档案设计为单一页面，不使用“档案设计 / 建档任务”双 Tab。

#### 老师建档路径

```text
教师手机端 → 班级 → 学生详情 → 学生成长档案
→ 新建档案
→ 选择已启用且适用当前年级的档案
→ 填写并保存草稿
→ 确认成档
```

#### 历史查看路径

```text
教师手机端 → 班级 → 学生详情 → 学生成长档案
→ 历史档案 → 档案详情
```

### 7.2 功能一：档案设计

#### 7.2.1 首页

首页只展示：

- `新建档案`主按钮。
- 学校已经创建的校本档案列表。
- 每份档案的名称、字段数和状态。

首页不展示：

- 推荐模板列表。
- 学生名单或学生搜索。
- 成档人数、完成率或任务统计。
- 填写人、提醒、截止时间等任务信息。

推荐模板只在“新建档案 → 从模板创建”中出现，避免与首页能力重复。

版本号只作为系统内部追溯和旧数据兼容信息，不在档案设计、学生填写、历史列表或档案详情中向老师展示。

#### 7.2.2 新建方式

点击“新建档案”后提供两种方式：

1. `完全新建`：创建不含分组和字段的空白草稿。
2. `从模板创建`：选择平台推荐模板，先预览，再点击“使用此模板”生成可编辑的校本草稿。

推荐模板本身不能直接修改。学校使用后形成独立校本档案，不受平台后续模板调整影响。

#### 7.2.3 基础设置

首期 MVP 只保留最小设置项，学校可以设置：

- 档案名称。
- 适用年级：一年级、二年级、三年级、四年级、五年级、六年级、七年级、八年级、九年级、高一、高二、高三。
- 自动带入字段：姓名、学号、性别、出生日期、年级、班级。
- 是否使用分组，以及档案字段。

首期不设置档案阶段和参与来源。一个学生拥有多份档案由“多档案规则”自然支持，不需要阶段概念区分。

自动带入字段使用“已选择 N 项”入口渐进披露，老师点击后在底部浮层中勾选。被选中的字段即为该档案确认成档所需的学生信息，不再单独提供“是否必填”开关。

建档教师和建档时间属于固定审计信息，不进入可选字段。姓名、性别、年级和班级等组织信息在档案填写页只读，不允许重复创建为手填字段。

#### 7.2.4 字段设计

档案字段配置复用平台统一表单构建器。分组是表单级布局开关，默认可以关闭；关闭时字段连续排列，不要求逐字段设置所属分组。开启后才展示分组管理和字段归组能力。

分组只保留名称，不设置分组说明。关闭分组不会删除已配置的分组关系，重新开启后恢复，避免误操作造成结构丢失。

支持：

- 新增分组时进入“添加分组”浮层，只填写名称，点击“完成”后才创建。
- 字段在卡片内展开编辑，字段类型、移动分组和删除等低频操作收进“更多操作”。
- 字段通过字段前的拖动把手调整顺序，支持鼠标、触控和键盘操作；字段展开区不展示上移、下移按钮。
- 开启分组时，字段拖动仅调整当前分组内顺序；跨组移动通过“所属分组”完成。
- 分组顺序仍通过编辑分组浮层中的上移、下移操作调整。
- 设置字段是否必填。
- 字段类型有五种：文字、单选、多选、日期、数字。文字不区分长短，由填写页统一提供多行输入；日期使用日期选择器，数字使用数字键盘输入。
- 编辑单选、多选选项。

开启分组时，每个字段必须属于一个分组；关闭分组时不校验字段所属分组。删除非唯一分组时，组内字段自动移动到剩余分组，不删除字段内容。

首期不设置“用于成长对比”。是否需要跨档案对比，在试点后按 8.3 评估。

#### 7.2.5 状态与操作

档案只有三种校本状态：

| 状态 | 学校可操作 | 老师是否可新建 |
|---|---|---|
| 草稿 | 编辑、保存、启用、删除 | 不可新建 |
| 已启用 | 禁用 | 可以新建 |
| 已禁用 | 重新启用、删除 | 不可新建 |

启用前必须满足：

- 名称不为空。
- 至少选择一个适用年级。
- 至少有一个字段。
- 开启分组时至少有一个分组，且所有字段已经归组。
- 首次开启分组时只展示“添加分组”入口，不自动创建或展示“默认分组”，也不展示分组数量；创建第一个分组后，已有字段自动归入该组。

已启用或已禁用档案不直接修改结构。若学校需要另一套结构，重新创建一份档案，避免已有草稿和历史内容发生歧义。

#### 7.2.6 删除规则

- 草稿可以删除。
- 已启用档案不能直接删除，必须先禁用。
- 已禁用档案可以删除；删除后从档案设计列表移除，不能重新启用，也不能再用于新建学生档案。
- 删除前必须二次确认，并明确提示此操作无法撤销。
- 删除档案设计不删除学生已有草稿和已成档记录。已有草稿仍可继续填写、保存和成档，历史档案仍可查看。
- 每份学生档案在创建时保存档案名称、布局、分组、字段和选项的不可变结构快照，并在系统内部保留版本标识。后续查看和填写使用该快照，不依赖档案设计是否仍然存在。

### 7.3 功能二：学生成长档案

#### 7.3.1 学生详情入口

学生详情的“成长报告”中增加“学生成长档案”入口。入口与期末报告等内容使用相同卡片层级，不新增顶部 Tab，避免挤压已有高频内容。

进入后展示：

- `新建档案`。
- 未完成草稿。
- 历史档案。

#### 7.3.2 选择档案

点击“新建档案”后，只展示同时满足以下条件的档案：

1. 学校创建的校本档案。
2. 当前状态为已启用。
3. 适用于当前学生年级。

草稿、已禁用档案和平台推荐模板不出现在选择列表中。

#### 7.3.3 填写和保存

- 档案设计中选定的学生信息由系统自动带出；建档教师固定带出。
- 学生资料已有值时直接显示。出生日期或学号缺失时，填写页显示“待补充”，老师可在当前页打开底部浮层补充并“保存并带入”，无需退出档案。
- 在档案填写页补充的出生日期或学号同步更新学生资料唯一数据源，并立即带入当前档案；不在档案草稿中维护一份独立副本。
- 字段按学校设计的分组和顺序展示。
- 默认展开第一个未完成分组，其他分组折叠。
- 历史档案可以在档案列表中随时回看。
- 老师可以随时保存草稿。
- 保存草稿允许自动带入字段或普通必填字段缺失。
- 确认成档时先校验自动带入字段，再校验普通必填字段；任一缺失都不能成档。
- 确认成档后，草稿转为不可直接修改的历史档案。
- 确认成档时保存自动带入字段的值快照。学生资料后续变化不回写、不覆盖历史档案。

#### 7.3.4 多档案规则

- 一个学生可以有多份档案。
- 不同档案分别保存，不合并为一个“是否已成档”状态。
- 同一份校本档案存在未完成草稿时，再次选择该档案优先打开原草稿，避免重复。
- 原草稿成档后，老师可以再次使用仍处于启用状态的档案新建下一份。
- 每份草稿和成档记录都在系统内部绑定创建时使用的档案编号和版本标识。
- 每份草稿和成档记录都保存创建时的档案结构快照，档案设计被删除后仍可独立展示。

#### 7.3.5 历史档案

历史列表按成档时间倒序展示：

- 档案名称。
- 所属周期。
- 成档老师。
- 成档时间。

历史档案只读。错误内容通过后续更正机制形成修订记录，不直接覆盖原记录。

### 7.4 启用与禁用边界

#### 启用

- 只表示该档案可以被老师用于新建。
- 不自动为任何学生创建档案或草稿。
- 不向老师发送通知或待办。
- 不要求老师完成填写。
- 不改变任何班级和学生权限。

#### 禁用

- 老师不能再使用该档案新建学生档案。
- 禁用前已经创建的草稿仍可继续填写和成档。
- 已成档的历史档案继续保留并可查看。
- 禁用不会删除档案设计、草稿或历史数据。
- 重新启用后，老师可以再次新建。

#### 删除

- 只允许删除草稿或已禁用档案设计。
- 删除是档案设计管理动作，不向学生档案数据级联。
- 删除后不再出现在档案设计列表和老师新建档案的选择列表中。
- 已有学生草稿和历史档案通过各自保存的结构快照继续填写和查看。

采用“已有草稿继续填写”的原因：草稿已经绑定学生、内容和填写过程。直接作废会造成老师已录入内容丢失，也会把模板管理动作错误地扩大为学生数据删除动作。

### 7.5 权限与信息安全

| 能力 | 学校领导/档案管理员 | 有学生访问权限的老师 | 平台工作人员 |
|---|---|---|---|
| 创建和编辑档案设计 | 可以 | 不可以 | 不可以 |
| 启用、禁用档案 | 可以 | 不可以 | 不可以 |
| 在档案设计中查看学生 | 不可以 | 不可以 | 不可以 |
| 新建和填写学生档案 | 仅同时具备该学生访问权限时 | 可以 | 仅在授权交付范围内写入草稿 |
| 查看学生草稿和历史 | 仅同时具备该学生访问权限时 | 可以 | 仅在授权交付范围内 |
| 确认成档 | 仅同时具备该学生访问权限时 | 可以 | 不可以 |
| 触发 AI 生成 | 不可以 | 不可以 | 可以，需学校授权 |

权限规则：

1. 档案设计权限由学校空间角色决定。
2. 学生档案权限沿用现有班级和学生访问关系。
3. 两套权限独立判断，任何一方都不能推导另一方。
4. 老师失去学生访问权限后，不能继续查看或填写该学生档案。
5. 完整档案查看、草稿保存、成档和状态变化需要留痕。

### 7.6 推荐模板

首期提供：

1. 一年级初始成长档案。
2. 学期成长档案。
3. 毕业与转衔档案。

建议用于教师交接的稳定字段包括：

- 优势特点。
- 兴趣倾向。
- 学习习惯。
- 情绪状态。
- 同伴交往。
- 当前关注。
- 有效支持方式。
- 阶段目标。

一年级初始档案参考结构：

| 分组 | 参考字段 |
|---|---|
| 学业基础 | 基础认知、专注习惯、提问与任务 |
| 兴趣偏好 | 兴趣倾向、动手创意 |
| 认知特点 | 学习方式、问题解决 |
| 交往风格 | 帮助分享、规则礼貌、冲突处理 |
| 性格与家庭 | 情绪稳定性、运动活力、主要照顾人、家庭陪伴时间 |
| 优先发展目标 | 家长期望、学生自选、教师建议 |
| 内驱力特征 | 当前突出内驱力信号、初始光芒定位、下一阶段优先关注方向 |
| 建档确认 | 家长确认 |
| 教师交接摘要 | 优势特点、当前关注、有效支持方式、阶段目标 |

### 7.7 AI 受控生成

首期学校端不展示 AI 生成按钮。

学校提出明确需求并授权后，平台工作人员可以基于问卷、访谈、教师观察或评价记录生成学生档案草稿。规则如下：

- 只能填写档案中已存在的字段。
- 没有资料依据的字段保持为空。
- AI 结果只能进入草稿，不能直接成档。
- 有学生访问权限的老师负责检查、修改和确认。
- 每次生成记录学校、档案、学生范围、资料来源、操作人和消耗。

### 7.8 核心数据对象

#### 档案设计

- 档案编号、学校编号和名称。
- 适用年级和状态。
- 已选择的自动带入字段。
- 分组、字段和选项。
- 内部版本标识、创建时间和更新时间。

#### 学生档案草稿

- 草稿编号和学生编号。
- 档案编号、名称、内部版本标识和结构快照。
- 字段内容。
- 创建人、创建时间和更新时间。

#### 已成档快照

- 学生编号。
- 档案编号、名称、内部版本标识和结构快照。
- 成档时的自动带入字段值快照和完整档案内容。
- 成档老师和成档时间。
- 修订关系。

#### 审计记录

- 操作类型。
- 操作人和角色。
- 学生编号或档案编号。
- 操作时间和结果。

### 7.9 异常与边界

| 场景 | 处理方式 |
|---|---|
| 学校没有已启用档案 | 选择页显示“当前年级暂无可用档案” |
| 档案缺少名称、年级、分组或字段 | 阻止启用，并提示缺少项 |
| 档案在填写期间被禁用 | 已有草稿继续填写；不能再创建新草稿 |
| 档案被重新启用 | 恢复新建能力，不影响原草稿和历史 |
| 同一校本档案已有草稿 | 打开已有草稿，不重复创建 |
| 同一学生有多种档案 | 分别保存和展示，不合并统计 |
| 两名老师先后打开同一草稿 | 两人看到同一份草稿；保存时按更新时间校验，发现旧版本时提示刷新后继续 |
| 老师中途失去学生权限 | 立即停止新的查看和保存请求，已保存数据保留 |
| 学生转班 | 新权限关系生效后，由新班级有权限老师继续访问；历史班级信息保留在快照中 |
| 学生离校 | 停止普通教师的新建和填写；历史按学校数据保留规则保存 |
| 自动带入字段在学生资料中缺失 | 允许保存草稿；出生日期或学号可在填写页补充并同步学生资料；补齐前不能确认成档 |
| 必填字段未完成 | 可以保存草稿，不能确认成档 |
| 已成档内容错误 | 生成更正草稿，原成档记录继续保留 |
| 单个学生 AI 生成失败 | 记录失败原因，不影响其他学生 |

### 7.10 产品假设

1. 学校愿意指定少量管理角色维护档案设计。
2. 老师更倾向于从学生详情按需建档，而不是处理统一待办。
3. 启用、禁用已经能满足首期学校对可用范围的管理需求。
4. 现有班级权限足以控制学生档案访问，无需增加档案专属班级权限。
5. 推荐模板可以覆盖多数学校的起步需求。

## 8. 发布计划

### 8.1 第一阶段：轻量化闭环

包含：

- 学校档案管理员入口。
- 完全新建和从模板创建。
- 推荐模板预览。
- 分组和字段设计。
- 自动带入学生字段选择。
- 文字、单选、多选、日期、数字五种档案字段。
- 草稿、已启用、已禁用三种状态。
- 学生详情档案入口。
- 按年级选择已启用档案。
- 填写、保存草稿和确认成档。
- 缺失出生日期或学号时在档案内补充并同步学生资料。
- 禁用后已有草稿继续填写。
- 历史档案查看。
- 模板状态与学生内容权限解耦。
- 关键操作审计。

不包含建档任务、责任人、截止时间、提醒、任务进度和待办角标。

### 8.2 第二阶段：数据安全与协作增强

根据试点结果决定是否增加：

- 多老师同时编辑的冲突提示和草稿变更记录。
- 更正草稿和修订版本页面。
- 更细的敏感字段访问控制。
- 学生转班、离校后的数据保留策略配置。
- 平台内部 AI 交付工作台。

### 8.3 后续候选能力

- 成长对比：填写新档案时参考上一份历史内容。只有试点学校明确提出跨档案对比需求时再评估。
- 只有学校出现明确、持续的统一催办需求时，才重新评估是否建设独立任务能力。该能力即使建设，也必须与档案设计保持独立入口、独立权限和独立状态，不能成为启用档案的必经步骤。

### 8.4 首期验收标准

1. 档案设计为单一页面，不出现建档任务 Tab。
2. 页面不出现全校成档率、已成档学生数或学生搜索。
3. 新建档案同时支持完全新建和从模板创建。
4. 推荐模板只在“从模板创建”中出现，选择后先预览再使用。
5. 空白草稿可以保存；缺少名称、年级或字段时不能启用；开启分组后还必须存在分组且所有字段已经归组。
6. 草稿可编辑；已启用档案只可禁用；已禁用档案只可重新启用。
7. 启用后不自动创建草稿、不产生提醒、不生成待办。
8. 学生详情提供“学生成长档案”入口。
9. 老师新建时只看到适用于当前年级的已启用校本档案。
10. 老师可以保存草稿和确认成档。
11. 同一校本档案已有草稿时打开原草稿，不重复创建。
12. 禁用后不能新建，但已有草稿仍可填写和成档。
13. 禁用不影响历史档案查看。
14. 一个学生可以保存多份不同档案。
15. 档案设计者不会因管理档案而自动获得学生内容查看权。
16. 学生档案访问继续沿用现有班级和学生权限。
17. “我的”页面和底部导航不出现建档任务待办或角标。
18. 页面不提供档案下载和转发。
19. 档案设计可选择姓名、学号、性别、出生日期、年级、班级作为自动带入字段，选中即为成档必填。
20. 学生资料已有值时直接带入；出生日期或学号缺失时可在档案填写页补充并同步学生资料，无需退出档案。
21. 自动带入字段或普通必填字段缺失时允许保存草稿，但不能确认成档。
22. 成档记录保存自动带入字段值快照，学生资料后续变化不影响历史档案。
23. 普通档案字段支持文字、单选、多选、日期、数字五种类型。
