# PRD-班主任助理本周行动建议

> 版本：V1.0  
> 更新日期：2026-07-14  
> 适用端：教师手机端  
> 当前阶段：领导汇报 Demo + 后续正式开发规则  
> 文档状态：待产品、研发、AI 服务与测试共同评审

## 1. 摘要

“本周行动建议”读取班级近期评价记录，帮助班主任快速找到当前最值得关注的学生、班级共性信号和本周可执行动作。功能采用自适应数据窗口：近期数据不足时向前累计，最多回看28天；同一班级同一行动周最多生成一份报告，重复进入直接查看已有结果。

当前 Demo 用于展示页面与内容效果，不代表已经接入真实班级数据和AI模型。正式版本需要按本文的数据、权限、生成、缓存和异常规则开发。

## 3. 背景

### 3.1 用户问题

班主任每天会看到不同老师提交的学生评价记录，但很难在有限时间内完成以下工作：

1. 从分散记录中找到连续出现的学生信号。
2. 判断哪些问题是个别事件，哪些是班级共性。
3. 关注其他老师发现、但班主任可能没有看到的学生亮点。
4. 把观察转化为少量、明确、可执行的班级管理动作。

已有统计报表适合查看数量和趋势，但不能直接回答“本周应该先做什么”。

### 3.2 为什么需要自适应数据窗口

不同班级的记录密度差异很大。固定只分析上一自然周，会出现两个问题：

- 数据多的班级可以形成有效洞察。
- 数据少的班级每周只能得到简单、重复、价值不高的报告。

因此，本功能把“建议执行周期”和“数据采集周期”分开：

- 建议执行周期：当前自然周。
- 数据采集周期：前1个、2个或4个完整自然周，根据数据充足度自动确定。

### 3.3 当前Demo状态

当前 Demo 已展示：

- 班主任助理入口。
- 本周行动建议加载状态。
- 数据充足时的完整报告。
- 往期建议列表与详情。
- 中等数据报告和前4周刚达到门槛的谨慎报告。

当前 Demo 尚未接入：

- 真实班级、评价记录和权限。
- 真实AI调用。
- 服务端生成任务、缓存和水位线。
- 正式错误、超时、排队和失败重试。

## 4. 目标

### 4.1 产品目标

1. 让班主任在3分钟内看懂当前最重要的班级信号。
2. 每份报告最多给出3条本周重点行动，避免信息过载。
3. 数据不足时不强行判断学生，也不生成低价值报告。
4. 从产品机制上避免同一批数据和同一班级报告被重复生成。
5. 在免费阶段控制模型文本消耗量，并支持平台规模化使用。

### 4.2 关键结果

| 关键结果 | 目标 |
|---|---:|
| 同一班级同一行动周的模型重复调用率 | 小于1% |
| 数据不足时误调用模型的比例 | 0% |
| 报告行动建议数量 | 不超过3条 |
| 报告正文长度 | 建议控制在700至1000个汉字 |
| 已生成报告重复访问的缓存命中率 | 不低于95% |
| 有权限用户查看历史报告成功率 | 不低于99% |

以上目标值为首版验收口径，试点后可以根据真实数据调整。

## 5. 用户与使用范围

### 5.1 核心用户

- 班主任：生成和查看本人带班班级的行动建议。
- 副班主任：在学校授权后查看同一班级报告，默认与班主任共享结果。

### 5.2 不在首版范围

- 普通任课教师独立生成班级行动建议。
- 自由聊天、连续追问和语音对话。
- 用户手动选择任意日期范围。
- 普通用户重新生成同一行动周报告。
- 实时预警和每条新记录到达后的自动更新。
- 跨班级、跨年级和跨学校综合分析。
- 自动把行动建议加入待办或考核。

## 6. 价值主张

### 6.1 对班主任

- 不需要逐条翻阅全班记录。
- 快速看到“需要留意、值得肯定、应该采取什么行动”。
- 能查看往期建议，避免同类问题反复处理却没有复盘依据。
- 数据不足时不生成报告，避免被不可靠结论误导。

### 6.2 对学校和平台

- 提高评价记录被实际使用的价值。
- 控制免费功能的模型成本。
- 通过共享报告、缓存和生成去重，避免重复消耗。
- 为后续行动跟进、班级管理复盘和付费能力积累数据基础。

## 7. 解决方案

### 7.1 核心概念

| 名称 | 定义 |
|---|---|
| 行动周 | 建议主要执行的当前自然周，周一00:00至周日23:59 |
| 数据窗口 | 本次报告实际读取的完整自然周，可为前1周、前2周或前4周 |
| 数据水位线 | 上一份有效报告已经分析到的最后一条记录编号或时间 |
| 新增数据 | 数据水位线之后、尚未用于正式报告的评价记录 |
| 有效报告 | 已完成AI分析并保存的行动建议 |
| 生成门槛 | 记录数量、学生覆盖率和活跃记录天数同时达到最低要求 |

### 7.2 页面结构

#### 7.2.1 班主任助理首页

入口：教师手机端 → 我的 → 管理工具 → 班主任助理。

首版提供两个方向：

1. 生成班级本周行动建议。
2. 诊断我的评价关注点。

“生成班级本周行动建议”的描述固定为：`根据近期评价，整理本周关注重点和跟进建议。`

班级选择规则：

- 只有一个带班班级时，点击后直接生成或查看已有报告。
- 有多个带班班级时，点击后打开“选择班级”底部弹窗。
- 点击班级即进入生成流程，不再增加确认按钮。
- 只展示当前教师拥有班主任权限的班级。
- 默认选中上次生成或查看的班级。

第二项未正式完成前，可以在 Demo 展示“能力建设中”；正式上线时应隐藏或明确标记为即将开放。

#### 7.2.2 本周行动建议页

页面顶部：

- 左侧返回按钮。
- 中间为页面标题，不展示助理形象。
- 标题下只展示班级名称和“根据{数据周期}评价记录生成”。
- 当前页标题“本周行动建议”已经说明建议适用于当前周，不再展示“建议周期”字段。
- 右侧使用历史图标进入“往期建议”。
- 生成等待阶段不展示数据周期、记录数、覆盖人数或参与老师数；报告生成完成后才展示数据来源。

生成等待阶段按实际数据窗口逐行展示：

- 前1周：正在整理上周评价记录 → 正在分析评价详情 → 正在归纳班级重点 → 正在生成本周行动建议。
- 前2周：增加“上周记录较少，继续参考前一周”，随后显示“正在分析前两周评价详情”。
- 前4周：继续增加“前两周记录仍较少，继续参考前四周”，随后显示“正在分析前四周评价详情”。
- 不显示百分比、骨架卡片和完成勾选；当前步骤使用轻微呼吸圆点。
- 以上步骤是等待反馈，不代表模型内部的精确执行节点；数据窗口扩展原因必须来自真实门槛判断。

报告完成后的数据来源：

- 前1周：`根据7月7日-13日评价记录生成`。
- 前2周：`上周记录较少 · 根据6月30日-7月13日记录生成`。
- 前4周：`前两周记录较少 · 根据6月16日-7月13日记录生成`。

数据充足报告最多包含：

1. 班级速览。
2. 学生洞察。
3. 评价洞察。
4. 班级洞察。
5. 本周重点行动，最多3条。

页面底部固定展示：`以上内容由AI基于评价记录生成，仅供参考`。

#### 7.2.3 往期建议页

- 页面名称：“往期建议”。
- 按建议开始日期归入对应年月，年月分组和组内建议均倒序排列。
- 月份使用“2026年7月”形式，左侧使用细竖线和小圆点建立时间层级，不增加状态或操作。
- 页面提供轻量班级切换；只有一个带班班级时显示普通班级名称，多个班级时点击名称打开共用的“选择班级”底部弹窗。
- 切换班级只改变当前历史浏览范围，不要求返回班主任助理重新进入生成流程。
- 列表只展示成功生成的报告，数据不足和生成失败不进入历史列表。
- 每条只展示精确的建议日期范围和“基于{数据周期}评价记录”；页面已表明为往期建议，列表不再重复“行动建议”。
- 不使用“第几周”命名建议，避免月初起算日和跨月周期的口径歧义。
- 不展示排序说明、摘要、记录数、覆盖人数和状态标签。
- 点击任一记录进入只读报告详情。
- 历史详情不再显示历史入口，避免页面循环跳转。

### 7.3 数据窗口规则

#### 7.3.1 首次生成

班级从未生成过有效报告时，系统按以下顺序检查：

1. 检查上一个完整自然周数据。
2. 不满足门槛时扩展到前两个完整自然周。
3. 仍不满足时扩展到前四个完整自然周。
4. 前四周仍不满足时不调用AI、不生成报告，也不写入往期建议。

不读取当前尚未结束的自然周，确保同一班级同一行动周的数据来源稳定。

达到门槛后立即停止扩展，不读取更早数据。

#### 7.3.2 后续生成

班级已有有效报告时：

1. 主要证据只读取数据水位线之后的新增记录。
2. 新数据不足时，不生成新报告，继续展示上一份有效报告。
3. 新数据继续跨周累计，最长累计28天。
4. 达到门槛后，生成面向当前行动周的新报告。
5. 上一份报告只提供固定字段的精简内容，用于判断问题是否持续；不得重复发送全部旧原始记录。

#### 7.3.3 同一周内达到门槛

- 周一数据不足时不生成报告，用户点击后只提示“近期记录较少，暂无法生成建议”。
- 周三新增数据达到门槛，允许在周三首次生成本行动周报告。
- 本周一旦生成成功，周四以后新增的数据进入下一次生成候选，不再更新本周报告。

### 7.4 数据充足度

首版建议使用可配置门槛，不把数值写死在前端。

建议同时检查：

- 记录数量。
- 学生覆盖率。
- 活跃记录天数。

初始参考值：

| 条件 | 建议起始值 |
|---|---:|
| 记录数量 | 不少于15条 |
| 学生覆盖率 | 不少于30% |
| 活跃记录天数 | 不少于3天 |

数据等级定义：

| 等级 | 规则 | 页面结果 |
|---|---|---|
| 充分 | 上一个完整自然周达到门槛 | 生成完整报告 |
| 中等 | 需要扩展到前2个完整自然周才达到门槛 | 使用统一字段生成克制报告 |
| 较少但可生成 | 需要扩展到前4个完整自然周才达到门槛 | 使用统一字段生成谨慎报告 |
| 数据不足 | 前4个完整自然周仍未达到门槛 | 不调用AI，不生成报告 |

不能只依赖记录条数。大量集中补录、只覆盖少数学生的记录，不应判断为数据充分。

### 7.5 内容生成规则

#### 7.5.1 输入

优先复用平台已有的结构化评价解析结果：

- 原始事件编号。
- 评价对象。
- 评价人。
- 记录时间和事件时间。
- 行为摘要。
- 匹配指标。
- 正向或负向。
- 分值。

不重复发送：

- 完整语音文件。
- 冗长语音转写原文。
- 已有统计图表。
- 与本次分析无关的上学期或更早记录。
- 同一原始事件产生的重复文本。

#### 7.5.2 输出

AI必须返回固定结构的JSON，不直接返回不可解析的长文章。所有成功报告字段完全一致，没有内容时返回空数组：

```json
{
  "class_overview": [],
  "student_insights": [
    {
      "student_name": "学生姓名",
      "content": "具体行为或连续信号",
      "suggestion": "面向班主任的建议",
      "need_verify": true
    }
  ],
  "evaluation_insights": [],
  "class_insights": [],
  "actions": []
}
```

内容要求：

- 只描述观察到的行为，不给学生贴标签。
- 数据中等时减少结论数量，明确需要核实。
- 记录少不等于学生表现少或存在问题。
- 推测性判断必须带“建议核实”。
- 行动必须具体，避免“加强教育”等空话。
- 学生姓名、班级和记录必须来自授权数据。

### 7.6 生成去重规则

#### 7.6.1 报告唯一键

服务端以以下字段确定同一份报告：

```text
学校编号 + 班级编号 + 行动周开始日期
```

同一唯一键最多存在一份成功报告。

#### 7.6.2 生成任务状态

以下状态只供后端控制生成任务，不属于报告JSON，也不在往期建议中展示：

| 状态 | 含义 | 用户看到的结果 |
|---|---|---|
| not_started | 未检查、未生成 | 可以发起检查 |
| insufficient_data | 前4个完整自然周仍未达到门槛 | 短提示“近期记录较少，暂无法生成建议” |
| queued | 已进入任务队列 | 等待生成 |
| generating | 模型正在生成 | 共享同一加载任务 |
| generated | 已生成 | 直接读取缓存报告 |
| failed | 生成失败 | 展示可操作错误和系统重试状态 |

#### 7.6.3 并发处理

- 多位有权限老师同时点击，只创建一个生成任务。
- 其他请求复用同一个任务编号并等待结果。
- 生成成功后所有人读取同一份报告。
- 生成失败允许系统自动重试，建议最多2次。
- 普通用户不能通过重复点击创建新任务。

#### 7.6.4 重新生成

首版不向普通用户提供“重新生成”。

以下情况可以由系统或管理员重新处理：

- 报告生成失败且自动重试仍失败。
- 数据错误被管理员修复。
- 模型输出无法解析或触发安全拦截。

重新处理仍复用原报告唯一键，并记录版本，不能额外创建多份同周报告。

### 7.7 数据水位线规则

每份成功报告至少保存：

- 本次使用的最早记录时间。
- 本次使用的最晚记录时间。
- 最大记录编号。
- 记录数量。
- 固定结构的报告内容。

下一次生成时，以最大记录编号或最后记录时间作为水位线。只有水位线之后的数据计入“新增数据”。

删除或修改历史记录时：

- 普通修改不自动重生成已发布报告。
- 修改后的记录进入下一周期分析。
- 重大数据错误由管理员触发修复流程。

### 7.8 模型成本约束

#### 7.8.1 调用约束

- 每班每行动周最多成功调用一次。
- 数据不足时不调用模型。
- 无人查看的班级不提前生成，首版采用首次访问时生成。
- 已生成报告重复访问全部读缓存。

#### 7.8.2 输入压缩

- 对同一原始事件去重。
- 使用短字段，不发送重复说明。
- 超过配置上限时保留完整统计，并按学生、正负向、指标和日期分层选取代表记录。
- 默认原始证据记录上限建议为120条，后续按模型和成本评估调整。
- 上一份报告只传固定字段的精简内容，不传全部旧记录。

#### 7.8.3 输出限制

- 输出最多1000个汉字。
- 学生洞察建议不超过3项。
- 本周重点行动最多3条。
- 避免重复复述记录数量和统计图表。

#### 7.8.4 平台保护

- 设置平台每日调用预算。
- 超过预算后进入排队或延迟生成，不重复提交任务。
- 监控单份报告输入量、输出量、生成时长、缓存命中率和失败率。

### 7.9 权限与隐私

- 用户只能查看本人有带班权限的班级报告。
- 班主任和已授权副班主任共享同一班级报告。
- 班级关系变化后立即重新校验查看权限。
- 报告包含未成年人信息，不允许公开分享链接。
- 服务端记录生成者、查看者、班级、时间和报告版本。
- 页面不展示模型内部提示词、成本、数据水位线等实现信息。

```text
content.class_insights
content.actions
```

生成任务状态通过独立接口返回，不混入报告内容。所有成功报告使用相同的 `content` 字段。

### 7.11 异常与边界场景

| 场景 | 处理方式 |
|---|---|
| 班主任没有带班班级 | 不展示生成入口，提示先完善带班信息 |
| 班主任有多个带班班级 | 进入前选择班级，记住上次选择 |
| 数据不足 | 不调用AI、不创建报告，短提示“近期记录较少，暂无法生成建议” |
| 正在生成 | 展示同一任务进度，禁止重复提交 |
| 网络中断 | 允许退出；再次进入读取任务状态 |
| 生成失败 | 展示稍后查看，不要求用户连续点击 |
| 输出解析失败 | 服务端重试或进入失败状态，不展示原始模型文本 |
| 权限被移除 | 立即禁止查看当前和历史报告 |
| 本周已经生成 | 直接打开报告，不显示重新生成 |
| 本周生成后新增记录 | 新记录进入下一次生成候选 |

### 7.12 Demo mock要求

当前汇报 Demo 固定展示：

1. 2025级四班：上周数据充分，展示前1周生成过程。
2. 2025级一班：上周数据较少，展示扩展到前2周的生成过程。
3. 2024级二班：前2周记录仍较少，展示扩展到前4周的生成过程。
4. 三个班级均提供成功生成的往期建议，用于展示历史班级切换。

Demo 中不展示数据等级切换器。三份历史报告使用相同JSON字段，不展示数据等级、摘要或状态标签。

### 7.13 验收标准

#### 页面与交互

- 本周行动建议右上角存在可点击的历史图标，触控区域不小于44×44像素。
- 图标具备“查看往期建议”的无障碍名称。
- 往期建议按执行周期倒序排列。
- 多班级教师点击生成入口时先选择班级；单班级教师不出现额外选择步骤。
- 往期建议可以在页面内切换带班班级，生成入口与历史页复用同一班级选择组件。
- 历史列表可进入中等数据报告和前4周谨慎报告。
- 历史详情返回后回到历史列表，再返回回到当前报告。
- 历史详情不重复显示历史入口。
- 当前报告顶部不展示助理形象和“建议周期”字段。
- 加载阶段不展示数据来源，报告完成后才显示。
- 加载阶段不展示记录数、覆盖人数、参与老师数和骨架卡片。
- 前1周、2周、4周分别展示对应的渐进分析文案。
- 历史卡片不展示摘要、统计和状态标签。
- 当前完整报告正文明显短于旧版，行动建议不超过3条。

#### 业务规则

- 同一学校、班级、行动周并发请求只创建一个任务。
- 数据不足时不调用模型。
- 已生成时重复访问只读取缓存。
- 本周生成后新增记录不触发本周重新生成。
- 水位线之后的新数据才能成为下一份报告的主要证据。
- 首次生成最多回看前4个完整自然周。
- 老师只能选择班级，不能手动选择分析时间范围。

#### 内容安全

- 中等数据报告不输出确定性学生标签。
- 前4周谨慎报告减少学生结论，并使用克制表达。
- 推测性结论提醒老师核实。
- 所有生成报告显示“以上内容由AI基于评价记录生成，仅供参考”。

## 8. 发布计划

### 阶段一：领导汇报Demo

### 阶段二：正式数据接入

- 数据充足度计算。
- 自适应7天、14天、28天窗口。
- 生成任务、缓存、唯一键和并发控制。
- 数据水位线。
- 历史报告存储和查询。
- 真实AI结构化输出。

- 上次报告延续判断。
- 生成排队和成本看板。

- 行动建议完成状态和后续复盘。
- 根据试点数据调整生成门槛。
