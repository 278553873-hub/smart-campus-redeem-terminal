# PRD-教师端问卷采集

> 版本：V1.2  
> 更新日期：2026-07-15  
> 适用端：教师手机端、家长手机端  
> 当前阶段：家长问卷与学生信息采集交互 Demo 已完成，待正式前后端开发  
> 文档状态：待产品、研发和测试评审

## 1. 摘要

问卷采集是面向学校多种信息收集场景的统一能力。本期支持两种模式：家长问卷由家长填写、每份答卷对应一名学生；学生信息采集由创建老师本人或各班班主任逐生填写，每名学生形成一条可保存草稿的独立资料记录。教师问卷预留底层能力，并在新建入口展示“即将开放”，本期不可创建。

本功能不使用 AI（人工智能）生成分析。所有统计均基于结构化答卷进行确定性计算，确保结果可解释、可复核。

产品采用人工控制采集状态的方式。家长问卷可以结束收集，未达到100%时允许重新开放；学生信息采集结束后允许恢复编辑，以满足资料纠错。分配给老师的学生采集通过应用内聚合待办触达，不按学生拆成大量待办。建议完成时间只用于家长问卷提示，不会自动阻止提交。只有草稿可以永久删除，已开始的采集任务通过结束和归档完成生命周期管理。

## 3. 背景与问题

学校常见的课后服务、研学活动、阅读习惯、校服尺码和家校需求调查，通常由老师在班级群中临时发起。现有方式存在以下问题：

1. 问卷与学生名单分离，老师难以确认哪些家庭已经完成。
2. 年级组长需要面向多个班级时，逐班、逐学生选择效率低。
3. 截止时间过于刚性，忙碌家长错过时间后无法补填。
4. 简答题回答数量较多时，全部堆在统计页会导致页面过长。
5. 问卷结束后缺少归档和恢复机制，列表会持续膨胀。
6. 删除规则不清晰，容易误删已经产生答卷的正式问卷。

此外，入学信息、健康信息和学籍核对等场景以学生为信息主体，但填写人是老师，无法直接套用“家长提交答卷”的模型。因此需要将底层升级为通用采集任务，同时形成家长问卷和学生信息采集两个业务闭环。

## 4. 产品目标

### 4.1 目标

1. 老师可以在手机端快速完成问卷创建和发布。
2. 默认面向全体学生，同时支持按班级或指定学生缩小范围。
3. 家长可以从待办直接填写，并在提交前完成必答校验。
4. 老师可以清楚查看完成情况、逐题统计和单份答卷。
5. 已结束问卷可以归档，低频数据不干扰日常使用。
6. 所有统计口径固定，不依赖 AI，前后端结果一致。
7. 创建人可以选择本人填写或按班级分配给班主任，老师可以按学生逐条填写资料、保存草稿和完成。
8. 教师可以从“我的”待办直达本人负责的聚合采集任务，完成本人负责的全部学生后待办自动消失。
9. 底层为未来面向教师发放问卷预留填写方、信息主体和任务分配能力。

### 4.2 非目标

V1 不包含以下能力：

- 不提供提醒未完成家长。
- 不自动催交，不发送二次提醒通知。
- 不包含微信服务通知等系统外消息通道，V1 以家长端待办作为送达结果。
- 不使用 AI 总结、生成结论或分析简答内容。
- 不支持问卷模板市场和公共题库。
- 不支持题目复制和复杂跳题逻辑；题目排序统一通过题目前的拖动把手完成。
- 不支持匿名问卷。
- 不支持家长修改已经提交的答卷。
- 不支持导出 Excel（电子表格）或 PDF（便携式文档格式）。
- 不支持已发布问卷直接编辑题目或发送范围。
- 不支持已发布问卷永久删除。
- 本期不开放教师问卷创建和填写。
- “我的”页面不展示没有待办的空入口。
- 学生信息采集不提供题目统计、答卷分析和量表题。
- 学生信息采集不支持图片、附件、地址级联和复杂字段联动。

## 5. 用户与权限

### 5.1 教师

- 创建、编辑和删除本人创建的草稿。
- 发布问卷到本人有权管理的学生范围。
- 查看本人创建问卷的数据和答卷。
- 结束收集、重新开放未满额问卷、复制、归档和恢复问卷。
- 创建学生信息采集任务，并按学生保存草稿或完成资料。
- 在确认开始时选择“我来填写”或“各班班主任”。
- 查看本人发起任务的整体进度和各班记录。
- 结束学生信息采集，并在需要纠错时恢复编辑。

### 5.2 年级组长等拥有多班权限的教师

- 候选班级和学生由现有组织权限服务提供。
- 页面默认选择“全体学生”，不要求老师逐班勾选。
- 需要缩小范围时，再渐进披露“按班级”和“指定学生”。
- 页面不展示权限推导过程，不增加“系统已根据权限生成范围”等说明。

### 5.3 学生信息采集填写人

- 当前老师被分配学生记录后，收到一个按采集任务聚合的待办，不按学生生成多条待办。
- 从“待我填写”进入时，只能查看和编辑分配给本人的学生记录。
- 本人负责记录全部完成后，待办自动完成并从待填写入口消失。
- 填写人不能结束、恢复、归档或复制任务，这些生命周期操作只属于任务创建人。

### 5.4 家长

- 查看与当前绑定学生相关、仍在收集且尚未提交的问卷。
- 预览题目、填写并提交答卷。
- 问卷已结束或该学生已经提交后，不得再次提交。

### 5.5 教师问卷填写人（后续版本）

- 校长或有权人员向教师范围发布问卷后，每名教师形成一条独立填写任务。
- 待填写问卷通过教师待办提醒，已提交记录可在“问卷采集－我收到的”查看。
- 本期只预留教师问卷的数据模型和禁用入口，不为教师问卷生成待办。学生信息采集的班主任待办属于本期能力。

### 5.6 权限原则

- 前端隐藏操作不等于完成权限控制，后端必须再次校验教师、空间、学生范围和问卷状态。
- 问卷归属当前学校或个人空间，跨空间不可查询或操作。
- V1 以学生为答卷主体。同一学生绑定多个家长时，任一家长提交后即视为该学生已完成，其他家长不可重复提交。
- 学生信息采集的目标范围仍受创建人组织权限约束。创建人选择“我来填写”时全部记录分配给本人；选择“各班班主任”时按发布时班级快照分配给对应班主任。
- 创建人可以查看任务全部学生记录；填写人只能查看和修改分配给本人的记录。
- 发布时没有可用班主任的班级不能完成“各班班主任”分配，必须先调整范围或补齐班主任关系。
- 底层必须分别校验填写人、信息主体和任务分配关系，不能只用用户编号代替三者。

## 6. 核心概念与数据口径

| 概念 | 定义 |
|---|---|
| 表单定义 | 标题、说明、布局模式、分组、字段、选项、必填规则等可复用内容结构 |
| 采集任务 | 一次面向指定范围发起的信息收集任务，包含模式、状态和范围快照 |
| 采集模式 | 家长问卷、学生信息采集或预留的教师问卷 |
| 信息主体 | 本条信息描述的对象；本期均为学生，未来教师问卷为教师 |
| 填写人 | 实际完成内容录入的人；家长问卷为家长，学生信息采集为教师 |
| 填写分工 | 学生信息采集的分配方式；分为创建人填写和各班班主任填写 |
| 聚合待办 | 同一采集任务分配给同一教师的全部学生记录合并成一个待办 |
| 对象记录 | 一个信息主体在一次采集任务中的独立状态与内容记录 |
| 目标学生 | 问卷发布时确定的接收学生快照 |
| 可送达学生 | 发布时已存在可用家长绑定关系、可以生成家长待办的目标学生 |
| 未送达学生 | 发布时没有可用家长绑定关系的目标学生 |
| 已完成 | 已成功提交一份有效答卷的学生 |
| 未完成 | 可送达但尚未提交答卷的学生 |
| 完成率 | 已完成学生数 ÷ 可送达学生数 |
| 建议完成时间 | 仅用于提示的可选时间，不是强制截止时间 |
| 有效回答 | 某道题实际存在回答的答卷数量 |
| 归档 | 将已结束问卷移出高频列表，保留数据并允许恢复 |
| 逐生草稿 | 学生信息采集中某名学生已部分填写，但尚未标记完成的记录 |

### 6.1 完成率规则

```text
可送达人数 = 目标学生中 reachable = true 的人数
已完成人数 = 成功提交答卷且学生在目标范围内的人数
完成率 = 可送达人数为0时取0，否则四舍五入为整数百分比
```

未送达学生不计入完成率分母。

学生信息采集整体口径：

```text
目标人数 = 采集范围内学生快照人数
已完成人数 = record_status = completed 的对象记录数
完成率 = 目标人数为0时取0，否则已完成人数 ÷ 目标人数
```

逐生草稿不计入已完成数，但退出后必须可以继续编辑。

学生信息采集待办口径：

```text
本人负责人数 = respondent_id 为当前教师的对象记录数
本人已完成人数 = 本人负责记录中 record_status = completed 的数量
待办存在 = 任务状态为收集中，且本人至少有1条未填写或草稿记录
待办数量 = 满足上述条件的采集任务数，不是学生记录数
```

### 6.2 发布快照

- 发布时固化目标学生、班级名称和可送达状态。
- 发布后新增、转班、离校或新绑定家长的学生，不自动改变本次问卷范围和完成率。
- 未满额问卷重新开放时沿用原目标快照和原答卷，不重新计算范围，不清空数据。
- 需要使用新名单时，老师应复制为新问卷并重新发布。

## 7. 产品流程

### 7.1 家长问卷

```text
教师创建草稿
  ↓
编辑问卷 → 选择发送范围 → 确认发布
  ↓
家长端生成待办并填写
  ↓
教师查看数据与答卷
  ↓
结束收集 ←→ 未满额时重新开放
  ↓
归档 ←→ 恢复到已结束
```

草稿可以永久删除；进入收集后的问卷不再允许永久删除。

### 7.2 学生信息采集

```text
教师创建草稿
  ↓
编辑采集表 → 选择学生范围 → 确认开始并选择填写分工
  ↓
系统按创建人或班主任生成填写分配
  ↓
填写人收到聚合待办 → 按学生保存草稿 / 标记完成
  ↓
本人负责学生全部完成 → 待办自动完成
  ↓
结束采集 ←→ 恢复编辑
  ↓
归档 ←→ 恢复到已结束
```

老师接收的是一个聚合采集任务，不生成与学生数量相同的教师待办。任务详情通过学生名单承载逐条记录。创建人从“我发起的”查看整体范围；填写人从“待我填写”进入时只查看本人负责范围。

## 8. 页面与交互需求

### 8.1 功能入口

发起入口：教师手机端 → 我的 → 更多工具 → 问卷采集。

填写入口：教师手机端 → 我的待办数量提示 → 待填写采集 → 待我填写。

- 更多工具保持一行4列。
- 问卷采集位于已有工具之后，不调整其他工具顺序。
- 顶部导航与科目管理、部门管理统一，导航高度44px。
- 当前教师存在待填写采集时，“我的”底部导航展示轻量数量提示，“我的”头像区下方展示紧凑的“待填写采集”入口；没有待办时两处都隐藏。
- 待办数量按采集任务计数，不按学生人数计数。

### 8.2 我发起的列表

“问卷采集”默认展示当前教师创建的任务。其他教师创建但分配给当前教师的任务不进入该列表。

页面右上角保留新建加号；存在待填写任务时，在加号左侧展示带数量的轻量待办图标，点击进入“待我填写”。两个图标各自保留不小于44px的触控区域。

顶部只保留3个高频页签：

1. 收集中。
2. 已结束。
3. 草稿。

不增加“已归档”作为第四个顶部页签。仅在存在归档数据时，在“已结束”页签顶部右侧展示轻量“已归档”二级入口，不展示归档数量，点击后进入独立归档列表。

#### 收集中和已结束卡片

展示：

- 问卷名称。
- 建议完成时间；未设置时展示“不限时间”。
- 短进度条。
- `已完成数/可送达数`，例如 `8/11`。
- 采集类型短文案：家长问卷或学生采集。

学生信息采集卡片的分母为目标学生数，分子为已完成的逐生记录数；不使用家长可送达口径。

不展示：

- “收集中”“已结束”等重复状态标签。
- “已提交”解释文案。
- 完成百分比数字。
- 展开箭头或“点击查看”说明。
- 发送对象和题目数量。

#### 草稿卡片

- 只展示问卷名称。
- 不展示草稿标签、题目数量和“继续编辑”。
- 点击卡片直接进入编辑页。

#### 待我填写

- 仅展示状态为收集中、至少有一条记录分配给当前教师、且本人尚未全部完成的学生信息采集任务。
- 每个任务只展示一张卡片，不按学生拆分。
- 卡片展示采集名称、创建人和`本人已完成数/本人负责人数`。
- 点击后只展示当前教师负责的学生，不能看到其他班主任负责的学生。
- 当前教师负责记录全部完成后，任务自动从“待我填写”和“我的”待办入口消失。
- 从“待我填写”进入详情时不展示结束、恢复、归档、复制等创建人操作。
- 选择“我来填写”的任务在未完成期间会同时出现在“我发起的”和“待我填写”，两处分别服务任务管理和内容填写。

### 8.3 新建和编辑问卷

#### 新建类型选择

点击右上角加号后，统一底部浮层展示：

1. 家长问卷：可点击，进入问卷创建流程。
2. 学生信息采集：可点击，进入采集表创建流程。
3. 教师问卷：禁用，展示“即将开放”。

浮层只使用一句短文案区分填写方，不放置业务规则说明。关闭浮层后不改变列表状态。

创建流程分为3步：

1. 编辑问卷。
2. 发送范围。
3. 确认发布。

三步底部操作保持一致：

- 左侧始终为“保存草稿”，可保存当前题目、发送范围、建议完成时间和学生采集填写分工。
- 右侧为“下一步”或“确认发布”。
- 顶部返回按钮在第2、3步返回上一步，不在底部重复提供“上一步”。

#### 第1步：编辑问卷

字段：

| 字段 | 必填 | 限制 |
|---|---|---|
| 问卷标题 | 发布必填 | 最多40个字符 |
| 问卷说明 | 否 | 最多120个字符 |
| 题目 | 发布必填 | 至少1题 |

- 新建问卷默认不创建任何题目。
- 题目标题行右侧始终只保留一个“添加题目”按钮。
- 题目为0时，不再在内容区重复展示“添加第一题”。
- 不提供复制题目功能。
- 保存草稿时允许标题或题目不完整，空标题保存为“未命名问卷”。
- 在发送范围和确认发布步骤也可以保存草稿，保存后返回草稿列表。
- 进入下一步前，必须完成标题和题目校验。

#### 题型

| 题型 | 配置 | 家长作答 | 统计方式 |
|---|---|---|---|
| 单选题 | 至少2个非空选项 | 只能选择1项 | 各选项人数和占有效回答比例 |
| 多选题 | 至少2个非空选项 | 可选择多项 | 各选项人数和占有效回答比例，比例之和可超过100% |
| 量表题 | 默认5级，可配置2至10级 | 选择1个分值 | 平均分、各分值人数和比例 |
| 简答题 | 无选项 | 输入文本 | 回答列表和确定性高频词统计 |

每道题支持设置“必答”。

题目编辑器交互规则：

- 问卷、学生信息采集和档案设计复用同一套表单构建器，字段卡片、分组、选项、必填、排序和删除交互保持一致。
- “使用分组”为表单级开关，默认关闭；关闭时题目连续排列且不展示所属分组，开启后才显示分组管理。
- 首次开启分组时只展示“添加分组”入口，不自动创建或展示“默认分组”，也不展示分组数量；创建第一个分组后，现有题目自动归入该组。
- 关闭分组不删除已有分组关系，重新开启后恢复；平铺期间新增的题目在重新开启时归入第一个已有分组。
- 选择题型后新增题目自动展开，并直接聚焦题目输入框。
- 题目和选项输入框使用白色编辑面、清晰边框和聚焦态，不得使用容易被理解为禁用的灰色输入框。
- 配置选项时使用中性序号，不展示单选框或复选框；老师此时是在编辑选项，不是在勾选答案。
- 每道题目前展示可操作的拖动把手，支持鼠标、触控和键盘排序；题目展开区不再提供上移、下移按钮。
- 开启分组时，拖动仅调整当前分组内的题目顺序；跨组移动通过“所属分组”完成。
- 量表级数使用减号和加号配置，达到2级或10级边界时禁用对应按钮。

#### 自定义答案选项

单选题和多选题支持增加“其他（请填写）”：

- 该能力收进题目编辑器的“更多”菜单，默认不展示，符合渐进披露。
- 添加后，该选项标记为“选中后需填写”。
- 家长只有选中该选项后，才展示补充文本框。
- 选中后补充文本为空时，不允许提交。
- 老师可以修改该选项名称，也可以删除该选项。
- 答案必须结构化保存选中项和补充文本，不得拼接成不可解析的普通字符串。

#### 第2步：发送范围

默认选择“全体学生”。老师只有需要缩小范围时，才切换到其他模式。

| 模式 | 行为 |
|---|---|
| 全体学生 | 选择当前教师有权管理的全部学生 |
| 按班级 | 支持多选班级，并提供全选和取消全选 |
| 指定学生 | 先切换班级，再选择学生，支持全选本班 |

规则：

- 不展示额外“已选择学生”摘要卡片。
- 未绑定家长的学生可以被选中，但发布确认时应显示未送达人数。
- 发送范围为空时不能进入发布。

#### 第3步：确认发布

展示：

- 标题和说明。
- 题目数。
- 目标学生数。
- 可送达学生数。
- 建议完成时间设置。
- 未送达人数提示，仅在大于0时出现。

建议完成时间：

- 默认关闭。
- 开启后默认填入7天后的20:00，老师可以修改。
- 发布后只作为提示，不会自动结束问卷。
- 到达建议完成时间后，问卷仍保持收集中，家长仍可提交。

#### 学生信息采集的第3步

学生信息采集仍使用同一套3步流程，不增加第4步。确认页展示采集名称、说明、字段数和目标学生数，并增加一行可点击的“填写分工”：

1. 我来填写：默认值，全部学生记录分配给任务创建人。
2. 各班班主任：按每名学生发布时的班级快照，分配给该班当前有效班主任。

点击“填写分工”后通过统一底部浮层选择，不在确认页直接铺开两种选项。选择“各班班主任”时：

- 同一班级的学生记录分配给同一名班主任。
- 创建人本人也是某班班主任时，该班记录正常分配给本人，并生成本人的聚合待办。
- 指定学生模式仍按学生所属班级分配，不要求创建人逐个选择填写老师。
- 任一目标班级缺少有效班主任时不能开始采集，页面只指出需要处理的班级，不展示组织权限推导过程。
- 开始采集后固化填写分配，班主任关系变化不自动改写历史任务。

### 8.4 草稿操作

编辑已有草稿时，顶部更多菜单提供“删除草稿”。新建但尚未保存的问卷不展示该入口。

删除前必须二次确认：

```text
删除草稿？
删除后无法恢复
取消 / 删除
```

只有草稿可以永久删除。后端收到删除请求时必须再次校验状态。

### 8.5 问卷详情

顶部基本信息使用一张卡片展示：

- 问卷名称。
- 问卷说明，最多显示2行。
- 建议完成时间，仅设置后展示。
- 预览入口。

不展示：

- 发送对象。
- “收集中”“已结束”等状态标签。
- 解释性摘要和操作说明。

详情页包含两个页签：

1. 数据。
2. 答卷。

不再拆分“概览”和“题目分析”。

### 8.6 数据页签

#### 完成情况

只展示一次整体完成情况：

- 已完成数/可送达数。
- 待提交人数。
- 未送达人数。
- 整体进度条。

每道题的统计卡片不再重复“共8份答卷”等整体信息。

#### 选择题和量表题

- 单选题、多选题展示各选项人数和比例。
- 自定义答案计入对应选项人数。
- 量表题展示平均分，保留1位小数，并按题目实际配置展示1至N级分布。
- 某题无人回答时，各项人数和比例均为0。
- 某题有效回答数与总答卷数不一致时，才展示“有效回答 N”。

#### 简答题

回答数量不超过5条时：

- 在题目卡片内全部展示。

回答数量超过5条时：

- 展示最多3个高频词及出现次数。
- 只展示最新2条回答，每条最多3行。
- 提供“查看全部回答”入口。

高频词必须使用确定性算法，不调用 AI：

1. 使用标准中文分词能力。
2. 去除停用词、标点、单字和空白内容。
3. 只统计出现次数不少于2次的词。
4. 按出现次数倒序，最多展示3个。
5. 相同次数使用稳定顺序，确保前后端结果一致。

“全部回答”页面：

- 默认按提交时间倒序。
- 支持搜索回答内容、学生姓名或学号。
- 存在多个班级时支持按班级筛选。
- 每次加载20条，支持继续加载。
- 展示回答内容、学生姓名、班级和提交时间。

### 8.7 答卷页签

提供3个筛选：

1. 已完成。
2. 未完成。
3. 未送达。

规则：

- 已完成列表展示学生姓名、家长关系和提交时间，点击进入答卷详情。
- 未完成列表展示可送达但尚未提交的学生。
- 未送达列表展示发布时没有有效家长绑定的学生。
- 不展示“已完成8”等重复数量。
- 不提供“提醒未完成家长”或“发送提醒”。

答卷详情按题目顺序展示题目、题型和格式化答案。自定义选项显示为：`其他：补充内容`。

### 8.8 预览问卷

- 问卷详情卡片提供“预览”。
- 预览必须复用真实家长问卷组件，避免预览与实际页面不一致。
- 预览模式可操作选择和输入，但不能创建真实答卷。
- 顶部提供“结束预览”返回问卷详情。
- 草稿暂不提供预览，V1 仅预览已发布或已归档问卷。

### 8.9 家长端待办与填写

家长端待办出现条件：

- 问卷状态为收集中。
- 当前学生在目标快照中。
- 当前学生发布时可送达。
- 当前学生尚未提交该问卷。

家长填写页展示：

- 问卷名称。
- 问卷说明，仅有内容时展示。
- 建议完成时间，仅设置后展示。
- 题目和必答标记。
- 提交按钮。

提交校验：

- 单选题必答时必须选择1项。
- 多选题必答时至少选择1项。
- 量表题必答时必须选择1个分值。
- 简答题必答时去除首尾空格后不能为空。
- 选中“其他（请填写）”时，对应补充文本不能为空。

提交时后端必须再次校验问卷仍在收集中。若老师在家长填写期间结束问卷，提交失败后提示：`问卷已结束或已经提交`。

提交成功后：

- 当前学生待办立即消失。
- 不允许重复提交。
- 老师端完成情况和答卷列表更新。

### 8.10 学生信息采集

创建流程：

```text
编辑采集表 → 选择学生范围 → 确认填写分工并开始采集
```

首版字段类型：

| 字段类型 | 输入与校验 |
|---|---|
| 单行文本 | 单行输入，适合姓名、证件号、地址等简短内容 |
| 多行文本 | 多行输入，适合健康情况、家庭情况等补充内容 |
| 数字 | 使用数字键盘，服务端校验数字格式 |
| 日期 | 使用日期选择控件，按标准日期格式保存 |
| 单选 | 至少2个选项，只允许选择1项 |
| 多选 | 至少2个选项，允许选择多项 |

点击“添加字段”后先展示单行文本、单选和日期；多行文本、数字和多选收进“更多字段”。学生信息采集不提供量表题。

采集详情：

- 基本信息卡展示采集名称、可选说明、完成进度和短进度条。
- 不展示采集对象摘要和“收集中”等状态标签。
- 默认展示学生名单，支持搜索，并按全部、未填写、草稿、已完成筛选。
- 每名学生对应一条对象记录，状态独立，不把全班学生拆成多个教师待办。
- 点击学生进入单页表单；未完成时可保存草稿，必填字段完整后可标记完成。
- 已完成记录在采集进行中仍可重新进入修改；保存后保持已完成状态。
- 采集结束后记录只读，通过“恢复编辑”重新进入可编辑状态。
- 学生信息采集不展示数据、答卷页签，不做题目统计分析。
- 创建人从“我发起的”进入时，完成进度和学生名单使用全部目标学生口径。
- 填写人从“待我填写”进入时，完成进度和学生名单只使用本人负责记录口径。
- 后端必须按填写分配过滤记录；不能只依赖前端隐藏其他班级学生。

### 8.11 教师端采集待办与教师问卷预留

学生信息采集的教师待办本期启用，采用两层结构：

1. “我的”底部导航在存在待办时显示红点或数量；进入“我的”后按条件展示紧凑待办入口。没有待办时整块隐藏。
2. “问卷采集”提供“待我填写”二级页面，用于查看本人尚未全部完成的聚合采集任务。

不新增第四个底部导航，也不在每次启动时强制弹窗。微信服务通知可作为后续外部触达渠道，点击后直接打开任务。教师问卷仍只预留 `respondent_type=teacher`、填写分配和禁用入口；本期待办只承载学生信息采集。

### 8.12 学生详情中的采集记录

学生详情使用三个并列页签：成长报告、评价记录、采集记录。学期选择和五育能力模型只放在“成长报告”中，切换到评价记录或采集记录后不重复展示。

采集记录列表规则：

- 同时展示家长问卷答卷和学生信息采集记录，统一称为“采集记录”。
- 只展示已经提交或标记完成的记录；未填写和草稿继续从“待我填写”处理，不作为正式学生资料展示。
- 按完成或更新时间倒序排列。
- 卡片展示采集名称、家长问卷或学生采集类型、完成时间和实际填写人。
- 不展示“已完成”、题目数、任务状态、说明或答案摘要。
- 已结束和已归档任务的完成记录继续展示。
- 点击卡片进入独立采集详情。
- 从采集详情返回学生详情时，保持在“采集记录”页签，便于连续查阅。

采集详情使用一张基本信息卡和一个连续问答列表：

- 基本信息卡展示采集名称、可选说明、采集类型、实际填写人和完成时间。
- 问答列表按表单字段顺序展示问题和格式化答案，不为每道题创建独立卡片。
- 可选字段没有填写时展示弱化的“未填写”。
- 选择题的自定义答案继续显示为“选项：补充内容”。

数据与权限规则：

- 按学生编号查询记录，不按学生当前班级查询；学生转班后历史记录仍然保留。
- 一次采集任务对一名学生最多展示一条完成记录。
- 任务创建人可以查看本人任务中的该学生记录；被分配教师只能查看明确分配给本人的学生采集记录。
- 学生详情不能绕过原采集任务权限展示健康、住址、家庭等敏感答案。
- 采集答案是信息来源，不自动覆盖学生基础信息。后续如需写入正式档案，必须增加字段映射和人工确认流程。

## 9. 生命周期与操作规则

### 9.1 状态机

家长问卷：

```text
草稿 --发布--> 收集中 --结束收集--> 已结束 --归档--> 已归档
  |                 ^          |                    |
  └--永久删除       └--重新开放┘          恢复到已结束
```

学生信息采集：

```text
草稿 --开始采集--> 收集中 --结束采集--> 已结束 --归档--> 已归档
  |                    ^          |                    |
  └--永久删除          └--恢复编辑┘          恢复到已结束
```

### 9.2 状态操作矩阵

| 当前状态 | 允许操作 | 禁止操作 |
|---|---|---|
| 草稿 | 编辑、保存、发布、永久删除 | 结束、重新开放、归档 |
| 收集中 | 查看、预览、复制、结束收集 | 编辑、永久删除、直接归档 |
| 已结束 | 查看、预览、复制、归档；未满额时可重新开放 | 编辑、永久删除；满额时重新开放 |
| 已归档 | 查看、预览、复制、恢复到已结束 | 编辑、永久删除、直接重新开放 |

### 9.3 详细规则

- 结束收集后，家长端立即停止接受新答卷。
- 重新开放保留原答卷、原范围和建议完成时间。
- 当已完成数等于可送达数且可送达数大于0时，问卷视为100%采集完成，不展示“重新开放”，后端也必须拒绝重新开放。
- 归档是可恢复操作，不需要二次确认。
- 恢复后进入“已结束”，不会直接进入“收集中”。
- 复制任何已发布问卷时生成新草稿，保留标题、说明、题目、范围模式和建议完成时间，清空答卷，标题增加“（副本）”。
- 已发布问卷不能永久删除。即使前端被绕过，后端也必须拒绝。
- 状态转换必须在数据库事务中校验旧状态，避免并发请求越级转换。
- 100%采集完成时禁止重新开放只适用于家长问卷；学生信息采集即使全部完成，仍允许恢复编辑。
- 学生信息采集结束后，所有填写人待办立即从待填写入口隐藏，逐生记录变为只读。
- 学生信息采集恢复编辑后，只为仍有未填写或草稿记录的负责人恢复聚合待办；本人负责记录已经全部完成的教师不重新出现待办。
- 复制学生信息采集时保留字段、范围模式和填写分工，清空全部学生内容并将对象记录重置为未填写；新任务开始时按新的组织关系重新生成填写分配。

| FR-22 | 学生信息采集支持“我来填写”和“各班班主任”两种填写分工 |
| FR-23 | 系统按任务和填写人生成聚合待办，不按学生拆分待办 |
| FR-24 | “我的”与底部导航仅在当前教师存在待填写任务时展示数量入口 |
| FR-25 | 待我填写详情只能查询和编辑当前教师负责的学生记录 |
| FR-26 | 当前教师负责记录全部完成后，聚合待办自动完成并隐藏 |
| FR-27 | 学生详情提供成长报告、评价记录和采集记录三个页签 |
| FR-28 | 采集记录按学生聚合家长答卷和已完成学生采集，并按时间倒序展示 |
| FR-29 | 采集详情使用连续问答列表展示完整答案，不自动覆盖学生基础信息 |

### 11.6 teacher_collection_todo

该表是教师聚合待办的读模型或缓存，不是逐生状态的唯一数据源。也可以由 `collection_assignment` 实时聚合，但必须保证查询性能和状态一致。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 聚合待办编号 |
| collection_task_id | string | 采集任务编号 |
| respondent_teacher_id | string | 被分配教师 |
| todo_status | enum | pending/completed/closed |
| assigned_count | integer | 本人负责记录数 |
| completed_count | integer | 本人已完成记录数 |
| updated_at | datetime | 最近更新时间 |
| completed_at | datetime/null | 本人全部完成时间 |

唯一约束：`collection_task_id + respondent_teacher_id`。

状态规则：

- 任务收集中且本人存在未完成记录时为 `pending`。
- 本人负责记录全部完成时为 `completed`，不再进入待填写查询。
- 任务结束或归档时为 `closed`。
- 任务恢复编辑时，仅仍有未完成记录的负责人恢复为 `pending`。
- 计数必须由逐生填写分配聚合更新，不允许前端直接写入。

### 11.7 collection_answer

| 字段 | 类型 | 说明 |
|---|---|---|
| assignment_id | string | 填写分配编号 |
| field_id | string | 字段编号 |
| answer_json | json | 结构化答案 |

推荐答案结构：

```json
{
  "selectedOptionIds": ["option-1", "option-3"],
  "customText": { "option-3": "补充内容" }
}
```

- 单选和多选使用以上对象；单选的 `selectedOptionIds` 只能有1项。
- 量表题直接保存数值，例如 `4`。
- 简答题直接保存字符串，例如 `"家长填写的回答"`。
- 单行文本、多行文本和日期保存字符串；数字保存数值或可精确解析的数字字符串。
- 学生信息采集保存草稿时允许缺少必填答案，标记完成时必须进行完整校验。

## 12. 接口建议

接口路径仅作为前后端评审基线，研发可以按现有服务规范调整命名，但不能改变状态和权限规则。

### 12.1 教师端

| 方法 | 路径 | 用途 |
|---|---|---|
| GET | `/teacher/collections?scope=created&status=&mode=` | 查询当前教师创建的任务列表 |
| GET | `/teacher/collection-todos?status=pending` | 查询分配给当前教师且本人未全部完成的聚合待办 |
| GET | `/teacher/collection-todos/pending-count` | 查询当前教师待填写采集任务数，用于“我的”数量提示 |
| GET | `/teacher/students/{studentId}/collection-records?cursor=` | 查询当前教师有权查看的该学生已完成采集记录 |
| POST | `/teacher/collections` | 创建草稿 |
| PUT | `/teacher/collections/{id}` | 更新草稿，非草稿返回409 |
| DELETE | `/teacher/collections/{id}` | 永久删除草稿，非草稿返回409 |
| POST | `/teacher/collections/{id}/publish` | 发布或开始采集，并按assignment_mode生成目标快照和填写分配 |
| POST | `/teacher/collections/{id}/end` | 收集中转已结束 |
| POST | `/teacher/collections/{id}/reopen` | 家长问卷重新开放或学生采集恢复编辑 |
| POST | `/teacher/collections/{id}/archive` | 已结束转已归档 |
| POST | `/teacher/collections/{id}/restore` | 已归档转已结束 |
| POST | `/teacher/collections/{id}/copy` | 复制为新草稿 |
| GET | `/teacher/collections/{id}` | 查询基本信息和表单字段 |
| GET | `/teacher/collections/{id}/statistics` | 查询家长问卷完成情况和逐题统计 |
| GET | `/teacher/collections/{id}/submissions?status=` | 查询家长问卷答卷名单 |
| GET | `/teacher/collections/{id}/submissions/{submissionId}` | 查询单份家长答卷 |
| GET | `/teacher/collections/{id}/fields/{fieldId}/responses` | 查询简答全部回答，支持search、class_id和cursor |
| GET | `/teacher/collections/{id}/student-records?status=&search=` | 查询学生信息采集逐生记录；创建人返回全部，填写人只返回本人负责记录 |
| GET | `/teacher/collections/{id}/student-records/{studentId}` | 查询某名学生的采集内容 |
| PUT | `/teacher/collections/{id}/student-records/{studentId}` | 保存逐生草稿或完成记录 |

接口权限规则：

- `scope=created` 必须按 `creator_teacher_id` 过滤，不能把分配给当前教师的其他人任务混入“我发起的”。
- 聚合待办接口只返回当前空间、任务收集中、本人存在未完成记录的学生信息采集任务。
- 待办卡片返回创建人、本人负责数和本人已完成数，不返回其他教师负责学生的明细。
- 逐生详情与保存接口必须校验当前教师是任务创建人或该学生记录的 `respondent_id`。
- 非创建人不得调用结束、恢复、归档、复制和删除接口。
- 开始学生采集时，`assignment_mode=homeroom` 且存在无法匹配有效班主任的班级，返回422和缺失班级编号，不创建部分任务。
- 学生采集历史接口必须复用原任务和填写分配权限，只返回已提交或已完成记录，并支持服务端分页。

### 12.2 家长端

| 方法 | 路径 | 用途 |
|---|---|---|
| GET | `/parent/students/{studentId}/questionnaires/pending` | 查询当前学生待填写问卷 |
| GET | `/parent/students/{studentId}/questionnaires/{id}` | 查询问卷题目和当前提交状态 |
| POST | `/parent/students/{studentId}/questionnaires/{id}/submissions` | 提交答卷 |

### 12.3 返回码约定

| 状态码 | 场景 |
|---|---|
| 400 | 参数格式错误 |
| 403 | 无空间、问卷或学生权限 |
| 404 | 问卷或答卷不存在 |
| 409 | 状态不允许、已经提交或版本冲突 |
| 422 | 题目、范围或答案业务校验失败 |

提交答卷接口应支持幂等键。网络重试不能产生重复答卷。

## 13. 异常与边界场景

| 场景 | 处理 |
|---|---|
| 草稿标题或题目不完整 | 可以保存草稿，不能发布 |
| 发送范围为空 | 禁止发布，保留当前选择 |
| 全部目标学生未绑定家长 | 允许发布；完成率为0，所有目标学生进入未送达列表，家长端不生成待办 |
| 建议完成时间早于当前时间 | 允许发布；发布后立即显示已到建议完成时间，但问卷仍保持收集中 |
| 建议完成时间已到 | 问卷继续收集，教师端显示逾期提示 |
| 家长打开后老师结束收集 | 页面可继续查看，提交时返回409并提示已结束 |
| 两位家长同时提交同一学生 | 数据库唯一约束只接受第一份，另一份返回409 |
| 老师连续点击结束或归档 | 接口幂等返回当前状态，不产生重复数据 |
| 已结束后重新开放 | 原答卷和完成率保留，未完成人员可以继续提交 |
| 已结束且已完成数等于可送达数 | 不展示重新开放；接口返回409，避免制造无法新增答卷的收集状态 |
| 已归档问卷恢复 | 恢复到已结束，不生成家长待办 |
| 问卷复制 | 生成独立草稿，不复制答卷和统计结果 |
| 题目无人回答 | 统计为0，不报错，不隐藏题目 |
| 简答回答超长 | 前后端统一限制长度，建议最多1000个字符 |
| 大规模全校问卷 | 列表、答卷和简答回答均由服务端分页，前端不得一次加载全部数据 |
| 学生信息采集保存不完整内容 | 允许保存为草稿；标记完成时返回422并定位缺失字段 |
| 学生采集已结束时继续编辑 | 保存接口返回409；老师恢复编辑后再保存 |
| 学生记录已完成后再次修改 | 采集中允许修改并保留完成状态，记录更新时间和操作者 |
| 选择各班班主任但某班未配置班主任 | 禁止开始采集，返回缺失班级；不得只生成部分分配 |
| 同一班主任负责多个班级或多名学生 | 仍只生成一个聚合待办，进度按本人全部负责记录计算 |
| 班主任关系在开始采集后发生变化 | 历史填写分配保持不变，不自动转交，避免任务范围和责任人漂移 |
| 填写人尝试查询其他班主任负责的学生 | 接口返回403，前端不展示该记录 |
| 填写人完成本人最后一条记录 | 聚合待办原子更新为完成，“我的”数量和待我填写列表同步减少 |
| 创建人结束学生采集 | 所有未完成聚合待办关闭，填写人已打开页面再次保存时返回409 |
| 创建人恢复学生采集 | 仅仍有未完成记录的负责人恢复待办，已全部完成的负责人不出现待办 |
| 学生采集记录仍为未填写或草稿 | 不进入学生详情采集记录页签 |
| 采集任务结束或归档 | 已完成记录继续在学生详情展示，未完成记录不展示 |
| 学生转班或离校 | 历史记录继续按学生编号关联，不随当前班级丢失 |
| 不同采集包含同名字段 | 分别保留在各自采集记录中，不自动合并或覆盖基础信息 |

## 14. 非功能要求

### 14.1 性能

- 问卷列表首屏接口建议在1秒内返回。
- “我的”待办数量接口建议在500毫秒内返回，可按教师和任务状态建立索引或使用聚合读模型。
- 统计接口在1万份答卷规模下建议在2秒内返回，可使用预聚合或缓存。
- 答卷列表和简答回答默认每页20条。
- 统计计算应由后端完成，前端只负责展示，避免大规模答卷传到手机端计算。

### 14.2 安全与隐私

- 教师只能查看授权空间内本人创建的任务，以及明确分配给本人的学生记录。
- 任务创建人可以查看任务全部目标记录；普通填写人不能查看其他填写人的记录和答案。
- 家长只能查看与本人绑定学生相关的问卷。
- 所有写操作记录操作者、时间、旧状态和新状态。
- 学生姓名、学号和家长关系属于敏感信息，接口日志不得完整输出答卷内容。

### 14.3 可用性与无障碍

- 手机端主要触控区域不小于44px。
- 列表、创建页、详情页和答卷页必须独立纵向滚动，不能因内容过多无法继续操作。
- 底部浮层使用统一组件，支持遮罩关闭、明确标题和无障碍名称。
- 状态反馈使用短提示，不在页面长期堆叠说明文案。

### 14.4 一致性

- 教师预览和家长真实填写复用同一题目渲染组件。
- 前后端共享题型、状态和答案结构定义，避免字符串口径不一致。
- 日期时间统一使用学校时区保存和展示，服务端存储建议使用 UTC（协调世界时）。

## 15. 验收标准

### 15.1 创建与发布

- [ ] 新建问卷时题目列表为空，不自动添加单选题。
- [ ] 题目为0时只展示题目标题行右侧的一个添加题目按钮。
- [ ] 单选、多选、量表和简答题均可创建、保存和发布。
- [ ] 量表默认5级，可配置2至10级，家长作答和统计按实际级数展示。
- [ ] 页面不提供复制题目能力。
- [ ] 自定义填写选项只通过“更多”入口添加。
- [ ] 默认范围为全体学生，按班级支持多选和全选。
- [ ] 建议完成时间默认关闭，关闭时问卷可以正常发布。
- [ ] 发送范围为空、标题为空或没有有效题目时不能发布。

### 15.2 家长填写

- [ ] 只有目标范围内、可送达、未提交且问卷收集中的学生出现待办。
- [ ] 必答题和自定义填写内容均完成后才能提交。
- [ ] 同一学生无法提交第二份答卷。
- [ ] 老师结束收集后，家长提交失败并看到明确提示。
- [ ] 建议完成时间到达后，老师未结束收集时家长仍可提交。

### 15.3 数据与答卷

- [ ] 完成率分母只使用可送达学生数。
- [ ] 完成情况只在数据页整体展示一次。
- [ ] 多选题比例允许总和超过100%。
- [ ] 简答不超过5条时全部展示，超过5条时只展示高频词和最新2条。
- [ ] 全部回答支持搜索、班级筛选和每次20条加载。
- [ ] 答卷页支持已完成、未完成和未送达筛选。
- [ ] 产品中不存在提醒未完成家长入口和相关接口。

### 15.4 生命周期

- [ ] 收集中问卷可以结束收集，但不能直接归档或删除。
- [ ] 未达到100%采集的已结束问卷可以重新开放、复制和归档。
- [ ] 已完成数等于可送达数且可送达数大于0时，已结束问卷不能重新开放。
- [ ] 已归档问卷可以恢复到已结束和复制。
- [ ] 只有草稿可以永久删除，删除前有二次确认。
- [ ] 已发布问卷不存在永久删除入口，后端也拒绝删除。
- [ ] 顶部始终只有收集中、已结束、草稿3个页签。
- [ ] 已归档通过已结束页中的二级入口访问。

### 15.5 页面质量

- [ ] 问卷列表不展示重复状态标签、百分比和点击说明。
- [ ] 问卷详情不展示发送对象和状态标签。
- [ ] 详情只有数据、答卷两个页签，不存在概览和题目分析重复页签。
- [ ] 多答卷、长问卷和全部回答页面均可以正常滚动到底部。
- [ ] 页面中不存在“以下为N份有效答卷的描述性统计”等解释性废话。

### 15.6 学生信息采集

- [ ] 点击新建后可选择家长问卷或学生信息采集，教师问卷不可点击并显示“即将开放”。
- [ ] 学生信息采集支持6种字段，常用字段优先展示，其他字段通过“更多字段”展开。
- [ ] 确认开始页默认展示“填写分工－我来填写”，点击后通过底部浮层切换为“各班班主任”，不增加第4步。
- [ ] 选择“我来填写”时全部学生记录分配给创建人；选择“各班班主任”时按班级快照正确分配。
- [ ] 任一目标班级没有有效班主任时不能开始采集，且不生成部分对象记录或待办。
- [ ] 选择范围后每名学生生成一条未填写记录，不生成多条教师待办。
- [ ] 学生名单支持搜索和全部、未填写、草稿、已完成筛选。
- [ ] 逐生表单可以保存不完整草稿，完成时校验全部必填字段。
- [ ] 已完成记录在采集中可以再次修改，保存后仍为已完成。
- [ ] 结束采集后记录只读；恢复编辑后可继续修改，即使完成率为100%。
- [ ] 学生信息采集不展示数据、答卷页签，不进入家长端待办。
- [ ] “我发起的”只展示当前教师创建的任务，其他人创建但分配给当前教师的任务只进入“待我填写”。
- [ ] 同一采集任务分配给同一教师的多条学生记录只生成一个聚合待办。
- [ ] “我的”底部导航和头像区下方仅在存在待填写采集时显示任务数量，无待办时完全隐藏。
- [ ] 待我填写卡片按本人负责范围展示进度，进入后只显示本人负责学生。
- [ ] 普通填写人无法查看其他班主任的学生，且不展示结束、恢复、归档和复制操作。
- [ ] 本人负责记录全部完成后，待办数量自动减少；任务从待我填写列表消失。
- [ ] 创建人结束采集后所有填写人待办关闭；恢复编辑时仅未全部完成的负责人重新出现待办。
- [ ] 教师问卷仍不可创建，不因学生采集待办能力而生成教师问卷待办。

### 15.7 学生详情采集记录

- [ ] 学生详情展示成长报告、评价记录、采集记录三个页签，文字在手机宽度内完整显示。
- [ ] 学期选择和五育能力模型只在成长报告页签展示。
- [ ] 采集记录同时包含有权限的家长问卷答卷和已完成学生采集记录。
- [ ] 未填写和草稿记录不进入学生详情。
- [ ] 记录卡片只展示名称、类型、完成时间和实际填写人，不展示重复完成状态和答案摘要。
- [ ] 点击记录进入独立详情，问答内容使用连续列表并可正常滚动到底部。
- [ ] 从采集详情返回后仍停留在采集记录页签。
- [ ] 已结束和已归档任务的完成记录仍可查看。
- [ ] 无权限教师不能通过学生详情查看其他任务或其他填写人的敏感答案。
- [ ] 采集答案不会自动覆盖学生基础信息。

## 16. 指标

首版上线后建议记录以下指标：

| 指标 | 说明 |
|---|---|
| 问卷发布成功率 | 进入确认发布后成功发布的比例 |
| 家长提交完成率 | 已完成学生数 ÷ 可送达学生数 |
| 平均完成时长 | 家长打开问卷到成功提交的时间 |
| 草稿继续编辑率 | 保存草稿后再次进入编辑的比例 |
| 结束后重新开放率 | 判断建议完成时间和结束机制是否合理 |
| 归档使用率 | 已结束问卷中被归档的比例 |
| 提交冲突率 | 因重复提交或状态变化返回409的比例 |
| 学生采集完成率 | 已完成逐生记录数 ÷ 目标学生数 |
| 逐生草稿继续率 | 保存逐生草稿后再次打开并完成的比例 |
| 恢复编辑率 | 已结束学生采集被恢复编辑的比例，用于判断纠错需求 |
| 班主任分工使用率 | 学生信息采集中选择“各班班主任”的任务比例 |
| 教师待办完成率 | 已完成聚合待办数 ÷ 已生成聚合待办数 |

不以“提醒点击率”为指标，因为 V1 不提供提醒功能。

## 17. 依赖与风险

### 17.1 依赖

- 教师空间和组织权限服务。
- 当前有效班主任关系和教师账号状态。
- 班级、学生和家长绑定关系。
- 教师端应用内待办入口和数量查询能力。
- 家长端待办或消息入口。
- 服务端分页、统计和幂等能力。
- 操作审计日志。

### 17.2 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 全校问卷答卷量大 | 手机端卡顿、统计接口超时 | 服务端分页、预聚合、缓存 |
| 发布后学生关系变化 | 完成率反复变化 | 使用发布快照，范围保持稳定 |
| 多家长重复提交 | 产生多份答卷 | 学生维度唯一约束和幂等提交 |
| 状态并发操作 | 问卷越级转换 | 事务校验旧状态和版本号 |
| 简答高频词误导 | 用户误认为 AI 结论 | 只展示词频和次数，不生成结论 |
| 已发布问卷误删 | 数据不可恢复 | 数据层禁止删除，使用归档代替 |
| 班主任关系缺失或错误 | 任务无法正确送达 | 开始前整体校验，返回缺失班级并阻止部分发布 |
| 聚合待办与逐生状态不一致 | 角标或进度错误 | 逐生记录作为事实来源，事务更新或可重建聚合读模型 |
| 发布后班主任发生变化 | 原负责人无法继续填写 | V1保持发布快照并记录审计，后续提供受控转交能力 |

## 18. 发布计划

### V1.2

- 教师创建、保存草稿和发布。
- 4种题型及自定义填写选项。
- 3种发送范围。
- 家长待办、填写和提交。
- 数据统计、答卷列表和详情。
- 结束、重新开放、归档、恢复、复制和草稿删除。
- 教师预览家长问卷。
- 功能统一命名为“问卷采集”，新建时选择采集类型。
- 学生信息采集创建、学生范围、6种字段和逐生记录。
- 逐生草稿、完成、搜索、状态筛选、结束和恢复编辑。
- 学生信息采集支持创建人填写和按班主任分工。
- “我的”待办数量、待填写采集入口和“待我填写”聚合任务列表。

- 问卷模板和常用题库。
- 数据导出。

- 定时发布。

- 学生信息采集图片、附件、地址级联、手动指定填写人和任务转交。

## 19. 当前 Demo 与正式开发差异

当前 Demo 使用浏览器本地数据模拟教师端和家长端联动，主要用于确认页面、信息层级和状态闭环。正式开发不能直接沿用本地存储方案，必须完成以下替换：

1. 表单字段、采集任务、对象记录、填写分配和答案写入服务端数据库。
2. 所有状态转换和删除规则由后端校验。
3. 家长待办从真实绑定关系和目标快照生成。
4. 统计由后端按统一口径计算。
5. 大规模列表使用服务端分页。
6. 提交接口实现学生维度唯一约束和幂等。
7. 增加权限、审计、异常监控和数据备份。
8. 学生信息采集使用服务端逐生记录接口，不以整份问卷 JSON 覆盖保存。
9. 学生信息采集待办接入真实教师账号、班主任关系和应用内待办服务。
10. 待我填写接口按当前教师服务端过滤逐生记录，不能只在前端过滤。
11. 教师问卷开放前复用统一教师待办底座，并补充教师作为信息主体的提交页面；本期教师问卷仍不创建待办。
12. 当前 Demo 使用固定的班主任 Mock（模拟）映射展示分工效果；正式开发必须使用真实班主任关系，并执行缺失班主任的整体发布校验。
13. 当前 Demo 在前端聚合学生采集历史；正式开发必须由服务端按学生、任务权限和填写分配过滤并分页返回。
