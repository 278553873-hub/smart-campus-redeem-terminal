# 家长手机端 UI 风格规范

本文档用于约束家长手机端 Demo 的视觉方向和页面设计。家长端应与教师端形成区分：教师端以橙色、蓝色为主，家长端以浅橙、浅黄、浅绿为主。

家长端服务对象是孩子家长，其中可能包含年龄较大的用户。因此视觉应温和、清晰、低信息密度，避免复杂说明和过度装饰。

---

## 1. 视觉定位

### 1.1 关键词

```text
温和、亲切、清爽、适老、低信息密度、浅橙、浅黄、浅绿、弥散光感
```

### 1.2 与教师端的区分

| 端 | 主色倾向 | 视觉气质 |
|---|---|---|
| 教师手机端 | 橙色 + 蓝色 | 高效、记录、管理、数据处理 |
| 家长手机端 | 浅橙 + 浅黄 + 浅绿 | 温和、陪伴、查看、家校共育 |

家长端不应沿用教师端的强管理感，也不应使用终端端的大屏炫光风格。

---

## 2. 背景使用范围

### 2.1 背景只覆盖 H5 内容区

家长端的弥散背景只应用在产品 H5 内容区，也就是手机屏幕内部的小程序页面区域。

禁止：

- 把整个浏览器页面、演示外层、手机壳外部背景都改成同一套弥散背景。
- 让 H5 背景影响 DEMO 控制台、真实手机壳、页面外框。
- 用背景色覆盖整个 `w-screen h-[100dvh]` 外层，让产品区域和演示环境混在一起。

推荐：

- 外层演示背景保持中性、低干扰。
- 手机壳外部可以是浅灰或项目默认演示底色。
- 只有 `PhoneMockup` 内部的 H5 页面使用家长端弥散背景。

### 2.2 背景职责

H5 背景只负责建立家长端气质，不承载业务信息。

背景不得：

- 抢走文字注意力。
- 造成按钮或卡片对比度不足。
- 使用复杂纹理、噪点、强渐变边界。
- 覆盖内容层级。

---

## 3. H5 背景色彩方向

### 3.1 主色调

家长端 H5 背景使用浅橙、浅黄、浅绿三类颜色。

推荐色值：

| 角色 | 色值 | 用途 |
|---|---|---|
| 浅橙 | `#FFD8B5` | 顶部或局部温暖光斑 |
| 浅黄 | `#FFF2B8` | 页面暖底、中心柔光 |
| 浅绿 | `#DFF6D6` | 成长、陪伴、舒缓氛围 |
| 近白暖底 | `#FFFDF2` | 内容区基础底色 |
| 浅绿白底 | `#F4FBF1` | 底部过渡或页面下半区 |

### 3.2 弥散效果

允许使用大面积、低透明、强模糊的弥散光斑。

推荐结构：

```text
顶部：浅黄 + 浅橙
中部：近白暖底
底部：浅绿过渡
局部：浅绿柔光点缀
```

推荐实现方式：

- 使用 `radial-gradient` 或绝对定位模糊圆形光斑。
- 光斑透明度低，边界必须柔和。
- 背景整体亮度偏高，避免沉重。

禁止：

- 高饱和橙色大面积铺底。
- 强荧光绿色。
- 蓝色作为家长端主背景色。
- 紫色、粉紫渐变等通用 AI 风格。
- 黑色或深色背景。

---

## 4. 内容卡片与背景关系

### 4.1 卡片承载信息，背景只做氛围

家长端主要信息必须放在白色或近白卡片中，不能直接铺在复杂背景上。

卡片建议：

| 类型 | 背景 | 圆角 | 阴影 |
|---|---|---:|---|
| 普通卡片 | `white/94` | 22px - 28px | 轻阴影 |
| 表单卡片 | `white` | 28px - 30px | 中轻阴影 |
| 当前孩子卡片 | `white/94` | 24px | 中轻阴影 |
| 底部导航 | `white/94` | 26px | 悬浮阴影 |
| 底部抽屉 | `white` 或 `#FFFDF7` | 顶部 32px | 上浮阴影 |

### 4.2 卡片与背景对比

- 背景是浅橙、浅黄、浅绿时，卡片必须足够白。
- 卡片内文字必须使用深色，不能使用浅灰作为主要信息。
- 阴影要轻，只表达层级，不制造厚重浮层。

---

## 5. 主色与功能色

### 5.1 家长端主色

| 角色 | 颜色 | 使用场景 |
|---|---|---|
| 家长端主操作 | 深绿色 / 墨绿色 | 绑定、确认等关键按钮 |
| 成长正向 | 绿色 | 表扬、成长、成功状态 |
| 温暖强调 | 橙色 / 浅橙 | 货币、重点数字、温暖提示 |
| 背景氛围 | 浅橙、浅黄、浅绿 | H5 页面背景 |

### 5.2 银行色彩

银行可以保留少量蓝色用于“存单、银行、金额操作”的识别，但不能成为家长端主色。

规则：

- 银行页可使用蓝色按钮或图标。
- 非银行页面不使用蓝色作为主强调色。
- 蓝色面积不能超过绿色和橙色的视觉权重。

---

## 6. 页面简洁原则

家长端必须继续执行强减法。

禁止出现：

- 页面标题下的灰色说明。
- “当前绑定的孩子”等重复标签。
- 报告列表简介。
- 评价正文直接堆在列表里。
- 老师、学校、学号等非当前任务必需信息同时出现。
- 兑换奖励、兑换商品、商城入口。

保留原则：

- 成长页只看最近表现。
- 报告页只找报告。
- 银行页只看货币、存款和存单操作。
- 孩子区域只帮助识别当前孩子。

---

## 7. 页面结构规范

### 7.1 登录页

- H5 内容区使用浅橙、浅黄、浅绿弥散背景。
- 只保留主标题和微信授权按钮。
- 不展示产品说明段落。

### 7.2 绑定孩子页

- 背景使用同一套家长端 H5 弥散背景。
- 表单放在白色卡片中。
- 不展示 Demo 说明。
- 不展示字段解释。

### 7.3 成长页

- 顶部展示当前孩子。
- 展示表扬次数、批评次数、净积分。
- 列表保留维度、时间、标题、分值。
- 不展示评价正文。

### 7.4 报告页

- 报告卡片只展示报告类型和时间。
- 不展示报告简介。
- 查看详情后再展示正文。

### 7.5 银行页

- 展示可用货币、银行存款。
- 存单流程参考货柜机，但视觉要更轻。
- 可保留“签署新存单 / 我的存单”。
- 不显示奖励兑换。

---

## 8. 修改检查清单

每次改家长手机端 UI 前检查：

- [ ] 弥散背景是否只在 H5 内容区内。
- [ ] 外层演示背景是否保持中性。
- [ ] 背景主色是否为浅橙、浅黄、浅绿。
- [ ] 蓝色是否只在银行等必要场景少量出现。
- [ ] 页面是否没有多余灰色说明。
- [ ] 报告列表是否没有简介。
- [ ] 当前孩子区域是否没有重复标签。
- [ ] 底部导航是否仍为成长、报告、银行。
- [ ] 家长端是否没有兑换、商城入口。
