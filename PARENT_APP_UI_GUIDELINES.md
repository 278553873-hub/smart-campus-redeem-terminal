# 家长手机端 UI/UX 设计规范

本规范仅适用于“家长手机端”微信小程序 Demo。家长端与教师手机端分别使用独立规范；当家长端页面与教师端规范存在冲突时，家长端页面优先遵循本规范。

家长端的核心用户不是学校管理员或教师，而是孩子家长，其中可能包含年龄较大的用户。因此页面必须优先保证：一眼看懂、少读文字、少做判断、少被装饰干扰。

---

## 1. 产品定位

### 1.1 使用场景

家长打开小程序，主要是为了快速确认：

- 孩子最近在学校表现怎么样。
- 有没有被表扬或被批评。
- 月度报告、期末报告是否可查看。
- 孩子当前有多少可用货币、银行存款是多少。
- 是否需要把可用货币存入积分银行。

### 1.2 设计目标

- 页面信息必须简洁、清晰、低压力。
- 每个页面只回答一个核心问题。
- 不用解释用户一眼能看懂的内容。
- 不把教师端、终端端、后台端的信息密度搬到家长端。
- 不为了“显得完整”增加备注、说明、标签、简介。

### 1.3 权限边界

- 家长端可以查看孩子成长信息。
- 家长端可以查看报告。
- 家长端可以查看可用货币、银行存款和存储记录。
- 家长端可以进行积分银行存储。
- 家长端不能兑换奖励、不能直接兑换商品。

## 2. 视觉基础规范

### 2.1 主色与辅助色

家长端采用温暖、清晰、低压力的色彩体系。

| 角色 | 色值 | 用途 |
|---|---|---|
| 主按钮背景 | `#FFC210` | 主要操作按钮、选中态重点背景 |
| 主按钮文字 | `#653C16` | 主按钮文字、底部导航选中态文字 |
| 辅助强调色 | `#5886EF` | 表单聚焦、次级强调、部分数据强调 |
| 页面背景 | 浅黄、浅橙、浅绿渐变 | 家长端整体柔和背景 |
| 内容卡片 | `#FFFFFF` 或接近白色 | 承载主要内容 |
| 主要文字 | 深灰或接近黑色 | 标题、重要对象、关键数字 |
| 辅助文字 | 中性灰 | 时间、来源、状态等弱信息 |

使用要求：

- 主操作优先使用 `#FFC210`，文字使用 `#653C16`。
- 次级强调可使用 `#5886EF`，但不应抢占主操作层级。
- 不使用高饱和大面积背景，不使用终端端炫光风格。
- 辅助灰字不能承载关键含义。

### 2.2 字体与字号

家长端必须优先照顾年龄较大的家长阅读体验。

| 信息类型 | 推荐字号 | 要求 |
|---|---:|---|
| 页面标题 | 17px | 清晰即可，不加副标题解释 |
| 重要对象名称 | 16px - 18px | 加粗，易识别 |
| 卡片标题 | 15px - 17px | 不使用过细字重 |
| 正文说明 | 14px - 15px | 仅必要时出现 |
| 辅助信息 | 12px - 13px | 不承载关键含义 |
| 核心数字 | 24px - 32px | 高对比、短标签 |
| 主要按钮文字 | 15px - 17px | 加粗、可快速识别 |

字号要求：

- 同一卡片内字号层级不超过 3 种。
- 关键操作文字不得低于 14px。
- 禁止大段 12px 灰字说明。
- 同类信息在不同页面应保持同一字号角色。

### 2.3 圆角、阴影与卡片

家长端视觉应柔和但不花哨。

| 组件 | 推荐圆角 | 要求 |
|---|---:|---|
| 顶部信息条、主要卡片 | 22px - 30px | 柔和、统一 |
| 表单输入框 | 16px - 18px | 明确可输入边界 |
| 主按钮 | 18px | 保持稳定识别 |
| 底部导航容器 | 24px - 28px | 与整体柔和风格一致 |
| 弹窗顶部圆角 | 28px - 32px | 与底部抽屉形态一致 |
| 小标签、小图标容器 | 12px - 18px | 不要过度装饰 |

阴影要求：

- 卡片阴影要轻，避免厚重悬浮感。
- 弹窗可使用更明显阴影，以建立层级。
- 不同页面的同类卡片应保持圆角、阴影、边框一致。

### 2.4 间距与密度

- 页面左右边距建议保持 20px 左右。
- 卡片内边距建议 14px - 20px。
- 卡片之间间距建议 12px - 20px。
- 首屏最多承载 1 个核心任务区和 1 个列表起始区。
- 不为了“呼吸感”牺牲核心信息读取效率。

### 2.5 2026-06 家长端轻科技视觉修订

本章节为 2026-06 家长端样式统一后的最新视觉基准；如与 2.1 或 2.3 中早期黄色主按钮、超大圆角建议冲突，以本章节为准。

家长端采用“极淡粉蓝/蓝绿背景 + 白色卡片 + 小面积渐变 icon”的轻科技视觉方向。

使用要求：

- 页面背景接近白色，只使用极淡粉蓝、浅蓝绿雾化氛围，避免多色大面积渐变造成脏感。
- 内容卡片以白色为主，使用轻阴影和克制圆角，不能所有组件都使用超大圆角。
- 功能 icon 使用统一渐变容器，青蓝用于报告、银行等信息入口，绿青用于成长、完成等正向状态，蜜桃橙用于待改进等关注状态。
- 待改进不使用灰色，避免误解为不可用或不重要。
- 确认类按钮使用青蓝/绿青渐变，不使用纯黑、深蓝或黄色大按钮。
- 圆角按组件层级分级：卡片、按钮、图标、底部抽屉分别控制，不允许局部临时写死成过度圆润风格。
- 学生头像真实上传优先，未上传时显示姓名姓氏第一个字，不使用随机卡通头像兜底。

## 3. 文案规范

### 3.1 少即是多

家长端文案必须克制。能通过标题、数字、列表结构理解的信息，不再追加说明文字。

不推荐：

```text
成长
最近评价与关键统计
```

推荐：

```text
成长
```

不推荐：

```text
报告
月度报告与期末报告
```

推荐：

```text
报告
```

不推荐：

```text
银行
可用货币、银行存款与存储记录
```

推荐：

```text
银行
```

### 3.2 禁止重复解释状态

如果界面已经通过结构表达了状态，不再增加说明标签。

不推荐：

```text
当前绑定的孩子
```

原因：用户能通过位置、选中态、页面内容理解当前状态，重复标签会增加视觉噪音。

### 3.3 报告卡片不展示简介

报告列表只展示必要识别信息，不展示内容简介。

报告卡片应包含：

- 报告类型。
- 时间范围。
- 状态或入口。

报告卡片不应包含：

- 总结段落。
- AI 摘要。
- 亮点列表。
- 灰色长说明。

AI 指人工智能。家长始终需要进入详情查看报告正文，列表页的任务只是帮助家长找到报告。

### 3.4 只在必要处解释

允许说明文字的场景：

- 首次未绑定孩子，需要解释为什么看不到数据。
- 表单错误或操作失败，需要告诉用户如何处理。
- 银行存储会改变可用货币和银行存款，需要在操作确认处说明影响。
- 涉及隐私、授权、数据安全，需要简短说明。

说明文字要求：

- 单句优先。
- 不超过 20 个中文字符为宜。
- 不连续出现两段说明。
- 不使用技术术语。

## 4. 交互与组件规范

### 4.1 标题栏

- 标题栏高度建议 44px。
- 标题居中，字号 17px，加粗。
- 标题栏只显示页面名，不加常规副标题。
- 有返回路径的页面必须提供明确返回按钮。

### 4.2 按钮

- 主要按钮高度不低于 48px。
- 主要按钮使用 `#FFC210` 背景和 `#653C16` 文字。
- 按钮文字应直接说明动作，不使用装饰性箭头。
- 禁用态使用低饱和灰色，避免误认为可点击。
- 按钮必须有触控按压反馈，例如轻微缩放或背景变化。

### 4.3 输入框

- 输入框高度建议 48px - 52px。
- 输入框边界必须清晰，聚焦态可使用 `#FFC210` 或 `#5886EF`。
- 表单字段应按真实填写顺序渐进展示。
- 表单错误必须说明如何修正，不只显示红色状态。

### 4.4 弹窗与浮层

- 弹窗、抽屉、浮层必须提供关闭方式。
- 关键流程中的弹窗必须提供返回或取消路径。
- 遮罩要保证底层内容不会干扰阅读。
- 弹窗中的主要操作保持单一，避免多个按钮抢主次。
- 弹窗内容应聚焦当前任务，不展示无关信息。

### 4.5 底部导航

家长端底部导航固定为 3 个入口：

```text
成长 / 报告 / 银行
```

禁止增加：

- 首页。
- 记录。
- 资产。
- 我的。
- 商城。
- 兑换。

原因：家长端的核心任务只有成长查看、报告查看、银行查看/存储。导航越多，年龄较大的家长越难判断从哪里进入。

## 5. 信息密度规范

### 5.1 一屏信息数量

每个手机首屏建议最多出现：

- 1 个当前上下文区域。
- 1 组核心统计或 1 个核心操作区。
- 1 个列表起始区域。

避免首屏同时出现成长、报告、银行、说明、入口卡片等多种任务。

### 5.2 卡片内容

卡片只保留决策必要信息。

评价记录卡片可包含：

- 五育维度。
- 时间。
- 标题。
- 分值。

报告列表卡片可包含：

- 报告类型。
- 时间范围。
- 查看入口。

银行记录卡片可包含：

- 存储或收益标题。
- 时间。
- 金额变化。
- 简短来源。

### 5.3 删除优先级

如果页面显得拥挤，按以下顺序删除：

1. 标题下灰色副标题。
2. 用户一眼能理解的解释文案。
3. 状态标签。
4. 卡片简介。
5. 装饰图标和装饰背景。

---

## 6. 禁止项

家长端禁止出现以下设计：

- 页面标题下的常规灰色解释副标题。
- “当前绑定的孩子”等重复状态标签。
- 报告列表中的内容简介。
- 兑换奖励、兑换商品主入口。
- 与当前页面任务无关的入口卡片。
- 教师端管理操作，如生成报告、导出全班、记录学生。
- 终端端强视觉风格，如大面积炫光、复杂动效、游戏化装饰。
- 大段说明文字堆叠。
- 让关键操作依赖小图标点击。

---

## 7. 评审清单

每次修改家长手机端前，必须检查：

- [ ] 页面是否只回答一个核心问题。
- [ ] 是否删除了用户一眼能看懂的备注说明。
- [ ] 顶部标题是否没有多余副标题。
- [ ] 是否没有重复状态标签。
- [ ] 报告列表是否没有简介。
- [ ] 底部导航是否仍为成长、报告、银行。
- [ ] 是否没有兑换奖励或兑换商品主入口。
- [ ] 主要按钮高度是否不低于 48px。
- [ ] 主要按钮是否使用 `#FFC210` 背景和 `#653C16` 文字。
- [ ] 辅助强调色是否优先使用 `#5886EF`。
- [ ] 同类卡片圆角、阴影、边框是否一致。
- [ ] 年级较大的家长是否能在 3 秒内理解当前页面要做什么。
