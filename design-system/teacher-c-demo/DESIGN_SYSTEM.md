# Demo C端改造教师端 Design System v1.0

> 定位：面向 `demo - C端改造 - 教师端` 的重构型设计系统，不继承现有 demo 的灰白低保真样式。
>
> 目标：建立一套年轻、轻科技、梦幻、温柔解压、低饱和，同时可支撑教师端真实业务复杂度的移动端规范。

## 0. 系统边界

### 适用页面

- 登录与授权：01 登录、微信手机号授权、手机号登录。
- 首次使用：02 完善信息、03 新手首页、创建班级、加入班级、添加学生。
- 高频记录：记录页、文字记录、拍照识别、AI 识别结果、编辑识别结果。
- 班级协作：班级列表、班级详情、老师列表、家长绑定、学生列表、批量编辑。
- 我的与设置：我的个人版、我的学校版、基本信息、设置、建议反馈、学科管理、部门管理、货币发放。
- 关键浮层：空间切换、邀请老师/家长、二维码邀请、微信转发确认、危险操作确认。

### 不适用页面

- PC 后台、运营后台、学校后台端、家长端正式页、货柜机终端。
- 这些端可以吸收色彩方向，但不能直接复用组件密度和移动端触控规则。

## 1. 设计性格

这套系统不做传统信息化，也不做旧 demo 的灰白线框。它的气质是“晨雾里的 AI 助教”：轻、聪明、柔软、可靠。老师使用时应感到清爽，不被系统打扰。

- 年轻：圆润、低压、轻快，图标和按钮有亲和力。
- 轻科技：AI 感来自透明度、细线、声波、微光和 IP，不来自强霓虹。
- 梦幻：允许极浅弥散背景和柔和渐变，但背景永远不抢内容。
- 温柔解压：错误、警示、危险都使用浅底和明确文案，降低紧张感。
- 高密度：真实内容优先，减少头像、解释文案和重复按钮。

## 2. Token 架构

`tokens.json` 是本系统的唯一 token 源文件，`tokens.css` 由 `scripts/build-tokens.mjs` 生成。组件和页面实现应优先引用 component token，其次 semantic token，避免直接使用 primitive token 或临时色值。

### 2.1 Primitive 原始色

| 色系 | Token | 日间 | 夜间 | 角色 |
|---|---|---:|---:|---|
| 晨雾青 | `--tc-aqua-500` | `#58C3CF` | `#72D7E2` | 主品牌、AI 识别、语音匹配 |
| 月光蓝 | `--tc-moon-500` | `#7F9EED` | `#9DB8FF` | 主渐变终点、链接、聚焦 |
| 鼠尾草绿 | `--tc-sage-500` | `#70BD9B` | `#92E2BF` | 成功、正向分值、通话中 |
| 雾紫 | `--tc-lilac-500` | `#AD94ED` | `#C3B1FF` | 空间切换、班级模式、梦幻氛围 |
| 花瓣粉 | `--tc-petal-500` | `#DF82AA` | `#F4AFCE` | AI IP 高光、温柔强调 |
| 奶油黄 | `--tc-cream-500` | `#E2B85D` | `#F2D184` | 提醒、待确认 |
| 珊瑚红 | `--tc-coral-500` | `#E77770` | `#FF9C95` | 错误、移除、掉线 |
| 墨蓝灰 | `--tc-ink-*` | 见 `tokens.css` | 见 `tokens.css` | 文本、背景、描边 |

### 2.2 Semantic 语义色

| Token | 用途 |
|---|---|
| `--tc-color-primary` | 每页主操作、选中态、默认强调 |
| `--tc-color-secondary` | 链接、辅助强调、焦点终点 |
| `--tc-color-accent` | 空间切换、班级模式、梦幻点缀 |
| `--tc-color-ai` | AI IP、AI 解读标题、智能生成标识 |
| `--tc-color-success` | 成功、已完成、正向分值 |
| `--tc-color-warning` | 待确认、需要关注 |
| `--tc-color-danger` | 删除、退出、错误、掉线 |
| `--tc-bg-page` | 页面底色 |
| `--tc-bg-card` | 卡片、弹窗主体 |
| `--tc-bg-soft` | 输入框、弱操作、分组背景 |
| `--tc-text-title` | 标题、姓名、关键数字 |
| `--tc-text-body` | 正文、字段值 |
| `--tc-text-muted` | 辅助、时间、占位 |

### 2.3 夜间模式

夜间模式是“温柔深色”，不是纯黑反色。

- 背景使用深蓝灰 `#121A2A` 到 `#101827`。
- 卡片使用深雾面，保留轻描边。
- 彩色提高亮度，但缩小面积。
- 警示色仍然使用浅透明红，不使用刺眼纯红。
- 阴影弱化，层级更多依靠描边与明度差。

## 3. 基础系统

### 3.1 圆角

| Token | 值 | 用途 |
|---|---:|---|
| `--tc-radius-xs` | `8px` | 小标签、小状态点底 |
| `--tc-radius-sm` | `12px` | 图标按钮、紧凑控件 |
| `--tc-radius-md` | `16px` | 输入框、标准按钮 |
| `--tc-radius-lg` | `20px` | 普通卡片、列表卡 |
| `--tc-radius-xl` | `24px` | AI 卡片、重点卡片 |
| `--tc-radius-2xl` | `28px` | 手机页 Hero、欢迎区 |
| `--tc-radius-sheet` | `30px 30px 0 0` | 底部弹窗 |
| `--tc-radius-pill` | `999px` | 胶囊、状态标签 |

### 3.2 阴影

| Token | 用途 |
|---|---|
| `--tc-shadow-card` | 普通卡片，轻微浮起 |
| `--tc-shadow-float` | 悬浮录入条、底部主操作 |
| `--tc-shadow-sheet` | 底部弹窗 |
| `--tc-shadow-danger` | 最终危险确认 |

规则：

- 阴影只表达层级，不表达装饰。
- 同一屏最多一个强浮起层。
- 列表项默认不用阴影，靠背景和描边区分。

### 3.3 描边

- 普通卡片：`1px solid var(--tc-border-subtle)`。
- 输入框：`1px solid var(--tc-border-control)`。
- 聚焦态：`1.5px solid var(--tc-border-focus)`。
- 危险态：`1px solid var(--tc-border-danger)`。
- AI 模块：使用青蓝或粉紫透明描边，不用高饱和描边。

### 3.4 图标

- 图标风格：线性、圆角端点、统一 1.75px-2px 线宽。
- 常规图标：18px。
- 底部导航：22px。
- 图标按钮：20px，热区 40px-44px。
- AI IP 不使用 emoji；可使用机器人头像、声波、星点等专属图形。

### 3.5 动效

- 页面进入：180ms 轻淡入 + 4px 上移。
- 底部弹窗：260ms 上滑，缓出。
- 按钮按压：缩放到 0.98 或背景轻微加深。
- 语音声波：1.1s 循环。
- 掉线：慢闪 2 次后停止，不持续刺激。
- 遵守系统减少动态设置，关闭非必要循环动效。

## 4. 字体系统

字体族：

```css
PingFang SC, Hiragino Sans GB, Microsoft YaHei, Noto Sans CJK SC, sans-serif
```

| 层级 | 字号 | 行高 | 字重 | 用途 |
|---|---:|---:|---:|---|
| Display | `26px` | `34px` | `700` | 02 页欢迎感标题、空状态主标题 |
| Page Title | `18px` | `26px` | `700` | 页面标题 |
| Section Title | `16px` | `24px` | `650` | 卡片组标题 |
| Card Title | `15px` | `22px` | `650` | 班级名、学生名、AI 标题 |
| Body | `14px` | `22px` | `400` | 正文、输入内容 |
| Body Strong | `14px` | `22px` | `600` | 关键字段值 |
| Meta | `13px` | `20px` | `400` | 时间、状态、辅助信息 |
| Label | `12px` | `18px` | `600` | 标签、筛选 |
| Micro | `11px` | `16px` | `500` | 极弱提示、角标 |

规则：

- 禁止大面积 `900` 字重。
- 列表页靠布局和色块分层，不靠全局加粗。
- 中文不使用负字距。
- 按钮文字最多 `700`，避免沉重。

## 5. 间距系统

| Token | 值 | 用途 |
|---|---:|---|
| `--tc-space-1` | `4px` | 图标文字微间距 |
| `--tc-space-2` | `8px` | 标签、小组件间距 |
| `--tc-space-3` | `12px` | 卡片内小组、列表项间距 |
| `--tc-space-4` | `16px` | 卡片内边距 |
| `--tc-space-5` | `20px` | 页面左右边距 |
| `--tc-space-6` | `24px` | 模块间距 |
| `--tc-space-8` | `32px` | 大空态、段落区隔 |
| `--tc-space-10` | `40px` | IP 与表单间距 |
| `--tc-space-12` | `48px` | 首屏核心区间距 |

触控规则：

- 可点击元素最小 44px。
- 列表行最小 56px。
- 底部主按钮高度 52px。
- 危险操作按钮不低于 48px。
- 页面底部有导航时，内容底部预留 96px-120px。

## 6. 通用组件

### 6.1 Button 按钮

| 变体 | 用途 | 规则 |
|---|---|---|
| Primary | 页面唯一主操作 | 青蓝到月光蓝渐变，高 52px |
| Secondary | 次操作 | 白色/深雾面，轻描边 |
| Soft | 弱入口、筛选 | 浅青或浅灰底 |
| Ghost | 取消、暂不处理 | 透明底，仅文字 |
| Danger | 退出、移除、删除 | 浅红底红字，最终确认可加描边 |
| Icon | 扫码、设置、更多 | 40px-44px 热区，只放图标 |
| Pill | 空间、筛选、状态 | 高 32px-36px |

规则：

- 一个页面最多一个 Primary。
- 主次按钮并列时，主按钮在右侧；上下排列时，主按钮在下方。
- 弱入口不写长文案，用图标+短词。

### 6.2 Input 输入

- 单行高度 48px。
- 多行最小 120px。
- 标签外置，字号 13px。
- 错误提示不超过 18 字。
- 聚焦态使用青蓝描边，不使用强阴影。
- 学号、班级号、手机号使用数字键盘。

### 6.3 Card 卡片

| 类型 | 用途 | 结构 |
|---|---|---|
| Plain Card | 普通信息 | 标题 + 关键字段 + 可选操作 |
| Action Card | 可点击入口 | 左内容 + 右箭头/图标 |
| AI Card | AI 解读 | AI IP + 标题 + 字段 + 总分 |
| Stat Card | 数据摘要 | 小标签 + 大数字 + 趋势 |
| Danger Card | 风险提示 | 短标题 + 一句说明 + 操作 |
| Person Card | 老师/学生/家长 | 头像/占位 + 姓名 + 标签 + 操作 |

规则：

- 卡片不嵌套卡片。
- 卡片不承载备注和逻辑推演。
- 卡片标题不超过 15px。
- 操作入口优先右上角图标或底部单行按钮。

### 6.4 Bottom Sheet 底部弹窗

用途：选择、编辑、邀请、更多操作、空间切换。

- 顶部圆角 30px。
- 遮罩日间 `rgba(17,27,43,0.38)`，夜间 `rgba(0,0,0,0.58)`。
- 标题 16px/24px/700。
- 内容最大高度 78%，内部滚动。
- 关闭按钮 40px。
- 选项行最小 48px。

### 6.5 Dialog 确认弹窗

用途：移除老师、移除家长绑定、退出班级、转移班主任最终确认。

- 宽度：屏幕宽度 - 48px。
- 标题 17px。
- 正文 14px，不超过 2 行为佳。
- 危险操作必须明确对象名称。
- 不可恢复操作使用分步确认，不把全部风险放在一个弹窗。

### 6.6 Navigation 导航

底部导航固定三个入口：记录、班级、我的。

- Active 使用 `--tc-nav-active`。
- 未选中使用 `--tc-nav-default`。
- 背景使用高不透明雾面白/深雾面。
- 不随记录学生/记录班级切换变色。
- 底栏高度 64px-72px，含安全区。

### 6.7 Segmented Control 分段控件

用途：记录学生/记录班级、未绑定/已绑定、学生/分组。

- 外层使用雾面浅底。
- 选中态用白色浮块 + 当前模式色短线。
- 字号 13px。
- 不使用强渐变填满整个控件。

## 7. 教师业务组件

### 7.1 AI 机器人 IP

用于 02 完善信息、AI 解读、空状态、生成中。

- 形象：聪明、可爱、可信，不能幼稚。
- 造型：圆润机器人头、小天线或耳机、明亮眼睛。
- 风格：线性 + 柔和填充，不做厚重 3D。
- 色彩：主体白/浅灰，眼睛青蓝，少量粉紫高光。
- 小尺寸头像：24px-32px，只保留脸部。
- 大尺寸展示：180px-220px，适合 02 页面。

### 7.2 Record Input Bar 悬浮录入条

结构：拍照 / 按住说话 / 文字。

- 容器：高 62px，左右 20px，底部导航上方。
- 背景：高不透明雾面白，青蓝弱描边。
- 中间按钮：宽度为左右入口的 1.8-2 倍。
- 拍照、文字使用线性图标，不用 emoji。
- 状态中不挤压记录流，必要提示放在录入条上方细条。

### 7.3 Record Card 记录卡

结构：

```text
时间 + 更多
原始输入
AI 智能解读
```

规则：

- 不展示当前登录老师头像和姓名。
- 原始语音先于转写文本。
- 原始文本默认最多 2 行。
- 点击卡片进入详情，更多菜单承接删除/复制等操作。

### 7.4 AI Interpretation Card

结构：

```text
AI IP + AI 智能解读 + 总分
时间
对象
指标
评语
```

规则：

- 整卡可点进入编辑，字段内不重复放编辑按钮。
- 总分是结果摘要，不像输入控件。
- 指标必须展示三级结构。
- 多指标按行展示，分值右对齐。
- 加分用鼠尾草绿，扣分用珊瑚红，和记录对象模式色分离。

### 7.5 Class Card 班级卡

结构：

```text
班级名 + 年级
任教关系标签
班级号 + 人数
学生列表 / 班级报告
更多
```

规则：

- 个人版可有创建/加入/显示设置入口，但收进右上角加号弹窗。
- 学校版不展示创建和加入。
- 班级号连续 8 位，不加空格。
- 卡片主视觉不使用橙色。

### 7.6 Class Detail Summary 班级详情摘要

- 标题格式：`2025级1班（一年级）`。
- 不展示班主任/语文等任教标签，这些属于关系，不属于班级基础信息。
- 仅班主任展示编辑入口。
- 班级号可复制。
- 个人版班主任可解散；学校版班主任不可解散，只能退出。

### 7.7 Teacher Card 老师卡

- 头像占位：姓名首字，圆形 44px。
- 信息：姓名 + 班主任/副班主任/学科标签。
- 班主任和副班主任置顶。
- 班主任视角展示更多操作；非班主任只读。
- 移除老师必须二次确认。

### 7.8 Student Card 学生卡

- 列表密度：一行可 2-3 个卡片，视页面而定。
- 展示：头像占位、姓名、性别、学号。
- 学生详情入口为整卡点击。
- 多选模式隐藏搜索框，展示全选、性别筛选、取消。

### 7.9 Parent Binding Card 家长绑定卡

- 未绑定：学生姓名 + 待绑定标签。
- 已绑定：学生姓名 + 家长关系，不在列表态展示手机号。
- 详情弹窗展示完整手机号和拨打电话。
- 编辑态才展示关系/手机号输入和移除绑定。
- 副班主任可管理时，视觉权限与班主任一致。

### 7.10 Space Switcher 空间切换

用于个人版、协作班级、学校版。

- 入口：记录页/班级页顶部弱胶囊；我的页信息区明显胶囊。
- 文案：`我创建的班级`、`李明老师的班级`、学校名称。
- 弹窗：列表展示空间名 + 类型标签 + 当前标记。
- 不展示“个人体验数据”等解释文案。

### 7.11 Invite Module 邀请组件

邀请老师与邀请家长共用结构，但品牌名称不同。

- 邀请老师：`AI素养评价`。
- 邀请家长：`素养指南针`。
- 弹窗选项：微信邀请、二维码邀请、链接邀请。
- 微信邀请为推荐项，但不要用强营销色。
- 二维码页提供保存图片。
- 链接邀请只展示可复制文案。

### 7.12 Management Tool Tile 管理工具卡

用于我的页：期末报告、学校数据报表、科目管理、部门管理、货币发放、建议反馈。

- 图标 20px，标题 14px。
- 可选一行辅助状态，不写解释。
- 一屏内工具卡统一高度。
- 学校版能力与个人版能力通过权限隐藏，不用灰掉一堆不可用项。

## 8. 页面模板

### 8.1 01 登录

- 顶部留白 + AI IP 小形象。
- 主按钮：微信授权登录。
- 次按钮：手机号登录/注册。
- 协议勾选放底部按钮下方，字号 11px。
- 不展示产品长介绍。

### 8.2 02 完善信息

目标：建立身份，轻松进入体验。

结构：

```text
顶部导航
AI 机器人 IP
姓名
学校
进入体验
```

规则：

- AI IP 宽 180px-220px。
- 字段只保留姓名、学校。
- 学校作为线索字段，不解释后台逻辑。
- 按钮不可用态只降对比，不改变位置。

### 8.3 03 新手首页

目标：让老师完成“获得班级、确认学生”两步。

- 仅保留 2 个步骤。
- 创建班级与加入班级并列卡片。
- 加入已有班级且已有学生时，确认学生自动完成。
- 首次记录引导放到记录页，不在首页增加第三步。

### 8.4 记录页

目标：最快完成记录、查看、修正。

- 顶部：记录学生/记录班级分段控件 + 指标弱入口 + AI 生成提示。
- 中部：记录流。
- 底部：悬浮录入条 + 底部导航。
- 空状态可展示 AI IP，但不写长说明。
- 记录流优先展示最近记录，减少装饰。

### 8.5 班级列表

- 个人版：右上角加号收纳创建、加入、显示设置。
- 学校版：只展示班级列表和筛选。
- 多空间用户显示空间切换胶囊。
- 班级卡底部操作只保留学生列表和班级报告。

### 8.6 班级详情

- 08A 个人版班主任：可编辑、转移班主任、解散班级。
- 08B 非班主任：只读，可邀请，可退出。
- 08C 学校版班主任：可编辑和转移班主任，不可解散。
- 老师列表和家长绑定列表只展示预览，完整管理进入独立页。

### 8.7 老师列表

- 班主任视角：可设置副班主任、取消副班主任、移除老师。
- 非班主任视角：只读，可邀请老师。
- 邀请按钮固定底部。
- 班主任/副班主任与其他老师之间用弱分割，不用重分组标题。

### 8.8 家长绑定列表

- 使用未绑定/已绑定分段控件。
- 未绑定学生优先展示。
- 已绑定列表按家长关系卡片展示。
- 列表态不展示手机号和操作图标。
- 点击进入详情弹窗，编辑态再展示可修改字段。

### 8.9 学生列表

- 默认：搜索 + 多选 + 学生卡片。
- 多选：全选、男、女、取消。
- 新增学生入口放列表下方，样式与学生卡一致。
- 批量编辑走独立页，不在列表内堆输入框。

### 8.10 我的页

- 14A 仅个人版：展示了解学校版入口，但不强销售。
- 14B 多版本：展示班级来源切换，不展示了解学校版。
- 管理工具按当前空间显示。
- 更多工具统一卡片，不把说明写进每个入口。

## 9. 状态系统

### 9.1 通用页面状态

| 状态 | 表达 |
|---|---|
| Loading 加载 | 骨架屏，保留页面结构 |
| Empty 空数据 | IP 小形象 + 一句引导 + 一个主操作 |
| Error 网络错误 | 浅红/浅黄提示 + 重试 |
| Denied 权限不足 | 说明原因 + 返回/联系管理员 |
| Submitting 提交中 | 主按钮 loading，防重复点击 |
| Success 成功 | 轻 toast 或按钮状态，不打断流程 |
| Disabled 不可用 | 降对比，保持布局 |

### 9.2 表单状态

- 空值：按钮不可用。
- 错误：字段下方短句。
- 保存中：按钮显示 `保存中...`。
- 保存失败：保留输入内容。
- 保存成功：toast 后返回或停留。

### 9.3 权限状态

- 不可操作的能力优先隐藏。
- 需要解释时，在弹窗或详情页给一句原因。
- 不在页面上铺满灰色禁用按钮。

### 9.4 AI 状态

| 状态 | 视觉 |
|---|---|
| 转写中 | 青蓝声波 + `语音转文字中` |
| 识别中 | AI IP 小头像 + `正在识别对象和指标` |
| 已入库 | 绿色标签 + 轻 toast |
| 需修正 | AI 卡片右上角弱提示 |
| 识别失败 | 浅红提示 + 手动补全入口 |

## 10. 语音状态

| 状态 | 色彩 | 图标 | 动效 | 文案 |
|---|---|---|---|---|
| 匹配中 | 晨雾青 | 搜索/波纹 | 1.6s 呼吸 | `正在匹配...` |
| 通话中 | 鼠尾草绿 | 麦克风/声波 | 声波跳动 | `正在听你说` |
| 静音 | 雾蓝灰 | 麦克风关闭 | 无循环 | `已静音` |
| 掉线 | 珊瑚红 | 断开/重连 | 慢闪 2 次 | `连接已断开` |

语音录入条状态：

- 默认：中间 `按住 说话`。
- 录音中：中间 `松手发送`，上滑取消。
- 转写中：按钮禁用，显示声波。
- 掉线：录入条上方出现紧凑提示，提供重新连接。

## 11. 版本与权限规则

### 11.1 空间类型

| 类型 | 展示名 | 视觉标签 | 可做 |
|---|---|---|---|
| 个人空间 | 我创建的班级 | 个人 | 创建/加入班级、记录、期末报告 |
| 协作空间 | 某某老师的班级 | 协作 | 查看/记录/邀请，按角色管理 |
| 学校空间 | 学校名称 | 学校 | 学校统一班级、学校报表、基础配置 |

### 11.2 角色

| 角色 | 权限表达 |
|---|---|
| 班主任 | 可编辑班级、管理老师、家长绑定、转移班主任 |
| 副班主任 | 可维护家长绑定，部分管理能力同班主任 |
| 任课老师 | 可查看、记录、邀请，不能编辑班级核心信息 |
| 学校版班主任 | 可维护班级协作，不可解散学校班级 |

规则：

- 页面骨架保持一致，差异只体现在操作入口。
- 危险操作按角色与空间决定，不使用同一按钮改文案硬套。
- 学校版不展示个人版营销入口。

## 12. 可访问性与可读性

- 正文与背景对比至少满足 WCAG AA（网页内容无障碍指南 AA 级）目标。
- 触控热区不低于 44px。
- 语音状态不能只靠颜色。
- 错误文案必须说明用户下一步能做什么。
- 夜间模式避免纯黑纯白高对比造成疲劳。
- 所有图标按钮需要可访问名称。

## 13. 禁止项

- 禁止被旧 demo 的灰白低保真样式限制。
- 禁止把备注、逻辑说明、状态推演写进真实页面。
- 禁止满屏高饱和渐变、强发光、重玻璃、厚重 3D。
- 禁止一页多个主按钮抢层级。
- 禁止普通页面复制 AI 解读卡视觉。
- 禁止用橙色表达加分。
- 禁止只靠颜色表达语音状态。
- 禁止在一个弹窗里堆所有风险说明；复杂风险必须分步披露。
