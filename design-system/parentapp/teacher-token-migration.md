# 家长手机端接入教师端 Design Token 迁移方案

> 目标：解决家长手机端当前视觉效果不佳的问题，用教师手机端的清爽浅色、AI 科技感、轻毛玻璃 token 提升质感，同时保留家长端“低压力、易读、少判断”的产品定位。

## 1. 迁移原则

家长端不直接复制教师端页面，而是复用教师端 Design Token（设计变量）的视觉语言。

```text
教师端 Design Token
        ↓
家长端语义映射
        ↓
家长端组件样式
        ↓
家长端页面落地
```

迁移时遵循三条边界：

- **只借用视觉质感**：浅蓝白背景、白卡、靛蓝/青色主色、轻毛玻璃。
- **不继承教师端信息密度**：家长端页面仍然一屏一个核心任务，不增加复杂筛选、图表或字段。
- **不继续使用暖黄色主操作**：家长端旧主色 `#FFC210` 在接入教师端 token 后降级为可选提醒色，不再作为主按钮色。

## 2. 本轮试点范围

先改“绑定孩子”页，不一次性改完整家长端。

选择绑定页作为试点的原因：

- 页面结构最简单，只有标题、输入框、按钮。
- 当前是家长端首屏，能最快判断视觉方向是否正确。
- 不涉及成长、报告、银行等复杂业务信息，不容易引入新的信息层级问题。

本轮不改：

- 成长页评价列表。
- 报告列表与报告详情。
- 银行页存单流程。
- 切换孩子弹窗。
- 底部导航样式。

## 3. Token 映射

| 家长端场景 | 使用教师端 token | 说明 |
|---|---|---|
| 页面背景 | `--tm-page-glow-left`、`--tm-page-glow-right`、`--tm-bg-page-gradient` | 替代暖黄/浅绿弥散背景，统一到教师端浅蓝白体系 |
| 卡片背景 | `--tm-card-bg` | 保持白卡，增强清爽感 |
| 卡片阴影 | `--tm-card-shadow` | 使用教师端轻阴影，避免旧家长端偏厚重阴影 |
| 输入框背景 | `--tm-input-bg` | 使用浅灰蓝输入区 |
| 输入框聚焦边框 | `--tm-input-border-focus` | 使用教师端主色，不再使用橙色聚焦 |
| 输入文字 | `--tm-input-text` | 保持高可读性 |
| 标签文字 | `--tm-text-muted` | 保留家长端易读的弱层级 |
| 主按钮背景 | `--tm-button-primary-bg` | 靛蓝到青色渐变，建立教师端一致感 |
| 主按钮文字 | `--tm-button-primary-text` | 白字，清晰明确 |
| 禁用按钮 | `--tm-button-disabled-bg` / `--tm-button-disabled-text` | 保持不可点击状态清楚 |

## 4. 绑定页试点设计

### 4.1 背景

旧方案：暖黄、浅橙、浅绿弥散背景。

新方案：教师端浅蓝白渐变，叠加浅粉和浅蓝弥散光。

目的：让家长端看起来与教师手机端属于同一产品体系，同时减少旧暖色造成的儿童化和营销感。

### 4.2 表单卡片

旧方案：白卡 + 较强阴影 + 暖色主按钮。

新方案：白卡 + 教师端轻阴影 + 靛蓝/青色主按钮。

目的：让表单更像现代小程序工具页，而不是促销活动页。

### 4.3 输入框

旧方案：输入框聚焦为橙色边框。

新方案：输入框聚焦为教师端主色边框。

目的：把注意力从暖色品牌转向统一的教师端 AI 科技色。

### 4.4 主按钮

旧方案：黄色按钮，文字为棕色。

新方案：靛蓝到青色渐变按钮，白色文字。

目的：强化“下一步/完成绑定”的主操作，同时与教师端主按钮一致。

## 5. 视觉验收标准

试点页完成后，浏览器验收时重点看：

- 学校编号输入框是否仍然一眼可见。
- 输入第二个字符时是否不再失焦。
- 学生姓名和学生学号是否在学校编号有内容后立即显示。
- 完成绑定按钮是否比其他元素更明显。
- 页面是否有教师端一致的清爽科技感。
- 字号、按钮高度、输入框高度是否仍适合家长手机端触控。

## 6. 试点通过后的铺开顺序

如果绑定页方向确认，再按下面顺序迁移：

1. 切换孩子弹窗：改卡片、按钮、遮罩。
2. 成长页：先改背景、统计卡片、筛选按钮，不动列表结构。
3. 报告页：改报告卡片和详情页容器，不增加摘要信息。
4. 银行页：改存单卡片和确认抽屉，保留银行流程文案。
5. 底部导航：确认是否继续复用教师端底部导航，或为家长端做低密度变体。

## 7. 风险与取舍

| 风险 | 影响 | 处理 |
|---|---|---|
| 家长端失去温暖感 | 可能显得偏教师工具化 | 保留简洁文案、大字号、白卡和柔和背景 |
| 主按钮从黄变蓝后识别变化 | 老用户可能感觉入口变了 | 试点阶段只改绑定页，确认后再铺开 |
| 教师端绿色报表色误用到家长端 | 视觉语义混乱 | 绑定页只使用 `primary`，不用 `class/report` 绿色 |
| 一次性全局替换颜色 | 容易破坏成长、银行等页面层级 | 分页面迁移，每次浏览器验收 |
